CSS

CSS describes how HTML is presented: which elements a rule targets, how boxes are sized and spaced, how they flow and stack, and how they respond to the viewport, their container, and user preferences. This page builds up concept by concept — from selectors and the box model through flexbox, grid, responsive and container queries, custom properties, animation, the cascade, logical properties, and finally practical architecture. Each runnable example has an editable HTML pane and an editable CSS pane; press Run to apply your changes in the live preview, or Reset to restore the original.

Selectors

A selector chooses which elements a declaration block applies to. The core building blocks are the type selector (p), the class selector (.card), the id selector (#hero), and the attribute selector ([type="email"]). These combine with pseudo-classes that match state or position (:hover, :focus-within, :nth-child()), pseudo-elements that target generated or sub-parts of an element (::before, ::after), and combinators that express relationships between elements.

The four combinators are descendant (a space), child (>), adjacent sibling (+), and general sibling (~). The forgiving :is() and :where() functions group selector lists so several patterns share one rule; they differ only in specificity, which the cascade section covers.

Note: attribute selectors support operators beyond exact match — ^= (starts with), $= (ends with), *= (contains), and ~= (space-separated word). Add i before the closing bracket, e.g. [href$=".PDF" i], for case-insensitive matching.

Note: ::before and ::after content is exposed to some assistive technologies but not reliably — never place meaning-critical text there. Decorative content: "" is announced by nothing, which is exactly what you want for pure decoration.

Box model

Every element is a rectangular box with four concentric layers: the content area, then padding inside the border, then the border itself, then margin outside it. Padding and border are part of what the element paints; margin is transparent space that separates boxes and, for adjacent vertical block margins, collapses.

The box-sizing property decides what width and height measure. With the default content-box, width sets only the content area, so padding and border are added on top and the rendered box is wider than the declared width. With border-box, width includes padding and border, so the rendered box matches the number you wrote — which is why most codebases set box-sizing: border-box globally.

Note: adjacent vertical margins between block boxes collapse to the larger of the two rather than adding. Padding never collapses, so if you need guaranteed space, padding is more predictable than margin.

Note: the near-universal reset *, *::before, *::after { box-sizing: border-box; } makes width math predictable and is safe to apply globally. Set it once at the top of your stylesheet.

Flow & positioning

In normal flow, block boxes stack vertically and inline boxes sit within lines of text. The display property changes participation: block takes the full width and forces line breaks, inline flows with text and ignores width/height, and inline-block flows inline while honoring width, height, and vertical padding. float pulls a box to one side and lets inline content wrap around it; clear pushes a later box below any floats.

The position property controls how a box is placed and whether it participates in flow. static is the default. relative keeps the box in flow but offsets it and establishes a containing block for descendants. absolute removes the box from flow and positions it against the nearest positioned ancestor. fixed positions against the viewport. sticky behaves as relative until a scroll threshold, then sticks. z-index orders overlapping positioned boxes within a stacking context.

Note: position: sticky is relative to the nearest scrolling ancestor and needs at least one inset (top, bottom, etc.) to know where to stick. A parent with overflow: hidden can silently break it.

Flexbox

Flexbox lays out items along a single axis. flex-direction sets the main axis — row (horizontal) or column (vertical) — and the cross axis is perpendicular to it. justify-content distributes items along the main axis; align-items aligns them on the cross axis. flex-wrap lets items flow onto new lines, and gap adds consistent spacing between them without margins.

Each item's flexibility comes from the flex shorthand: flex-grow (share of leftover space), flex-shrink (how it gives up space when cramped), and flex-basis (its starting size before growing or shrinking). flex: 1 is the common shorthand for "grow to fill equally".

Note: align-items sets cross-axis alignment for every item; align-self overrides it for one. A single grow item is the standard trick to pin a card's footer to the bottom regardless of body length.

Grid

Grid lays out content in two dimensions at once. grid-template-columns and grid-template-rows define the tracks. The fr unit distributes leftover space, repeat() shortens repetitive definitions, and minmax() sets a floor and ceiling for a track. gap spaces tracks apart.

grid-template-areas lets you name regions and paint the layout as ASCII art, which is exceptionally readable for page shells. When items land outside the explicit grid, grid-auto-rows and grid-auto-flow govern the size and direction of the implicit tracks that get created for them.

Note: the strings in grid-template-areas must form a complete rectangle — every row needs the same number of cells, and each named area must be contiguous. A single dot . marks an empty cell.

Note: auto-fill keeps empty columns when content runs out; auto-fit collapses them so existing items stretch to fill the row. Swap one for the other to see the difference on a wide preview.

Responsive design

Mobile-first design writes the base styles for small screens, then layers enhancements with min-width media queries as more space becomes available. This keeps the default lightweight and progressively adds columns and spacing. A fluid image simply uses max-width: 100%; height: auto so it never overflows its container.

Media queries also read user preferences, not just dimensions. prefers-reduced-motion lets you strip animation for users who ask for it, and prefers-color-scheme adapts to the system light or dark setting. Designing against preferences is as important as designing against breakpoints.

Example: mobile-first reflow and a fluid image

Note: media queries only respond to real viewport width if the document sets <meta name="viewport" content="width=device-width, initial-scale=1">. Without it, mobile browsers report a wide virtual viewport and your breakpoints never fire.

Note: toggle your operating system's dark mode and reduce-motion settings to see both queries respond. Preference queries make the interface respect choices users have already expressed system-wide.

Variables & calc()

Custom properties (CSS variables) are declared with a --name syntax and read with var(--name). They cascade and inherit like normal properties, so a value set on :root is available everywhere, while a value set on a component scopes to that subtree. var() accepts a fallback as its second argument — var(--gap, 1rem) uses 1rem if --gap is not defined.

Math functions compute values at used-value time. calc() mixes units (calc(100% - 2rem)); min() and max() pick the smaller or larger of several values; and clamp(min, preferred, max) bounds a fluid value between a floor and a ceiling. Combined with custom properties, these express responsive spacing and typography without media queries.

Example: a themeable component that toggles a variable

Note: custom properties inherit, so overriding --accent on .brand re-themes everything inside it in one line. This is the foundation of runtime theming — a single JavaScript element.style.setProperty('--accent', color) repaints the whole subtree.

Example: fluid spacing and type with clamp() and calc()

Note: calc(), min(), max(), clamp(), and custom properties are supported in every current browser. The one gotcha: you cannot concatenate a bare number with a unit inside var() — write calc(var(--n) * 1px), not var(--n)px.

Transforms & animations

The transform property moves, scales, rotates, and skews an element without disturbing surrounding layout: translate(), scale(), rotate(), and skew(). transform-origin sets the pivot point — the default is the element's center. Because transforms are applied on the compositor, they are among the cheapest properties to animate.

transition interpolates a property between its old and new value over a duration whenever that value changes, which is ideal for hover and focus states. @keyframes plus the animation shorthand define multi-step, self-running motion. will-change hints that a property is about to animate so the browser can promote the element to its own layer — use it sparingly, as over-promoting wastes memory.

Note: animate transform and opacity whenever possible — they run on the compositor and skip layout and paint. Animating width, top, or margin forces reflow on every frame and drops frames on weaker devices.

Note: any looping or attention-grabbing animation should be silenced under prefers-reduced-motion: reduce. Continuous motion can trigger nausea or migraines for some users, so treat the still state as a first-class variant, not an afterthought.

Container queries

Media queries respond to the viewport, but a component often cares about the space it was given, not the whole page. Container queries fix that. Declare an element a query container with container-type: inline-size (optionally naming it via container-name), then write @container rules that respond to that container's width. The same component can render compact in a sidebar and expanded in a main column with no knowledge of the page layout.

@container conditions use logical size features like min-width and max-width, mirroring media query syntax. Because queries target the nearest ancestor container, components become genuinely portable: drop them anywhere and they adapt to their slot.

Note: container queries are Baseline across current Chrome, Edge, Firefox, and Safari. For older engines, the base (column) styles are the natural fallback — write the compact layout as the default and treat the row layout as the enhancement, exactly as above.

Specificity & cascade

When several rules target the same element, the cascade decides which declaration wins. Origin and importance come first, then specificity, then source order. Specificity is a three-part count: id selectors, then class/attribute/pseudo-class selectors, then type/pseudo-element selectors. A higher-weight column always beats a lower one regardless of how many lower-weight selectors pile up — one id (1,0,0) outranks ten classes (0,10,0).

!important jumps a declaration above normal ones but creates escalation wars, so it is a last resort. Better tools: :where() contributes zero specificity, so you can group selectors without raising weight, and @layer establishes explicit precedence between named layers so later layers win no matter the selector strength inside them.

Note: the real lesson is upstream — avoid styling by id in the first place. If the base rule were .btn and the modifier .btn--danger, source order alone would resolve it and there would be no conflict to solve.

Note: @layer is supported across all current browsers and is the modern replacement for specificity hacks. Unlayered styles always outrank layered ones, so keep third-party resets in a low layer and your own components above them.

Logical properties

Physical properties like margin-left and top are tied to screen directions. Logical properties instead reference the flow-relative axes: the inline axis runs along the text direction, and the block axis runs across lines. margin-inline sets both inline sides, padding-block sets both block sides, and inset-inline-start means "the leading edge" whatever the writing direction.

Because these follow direction and writing-mode, a layout authored with logical properties mirrors automatically for right-to-left languages and rotates for vertical scripts — no separate RTL stylesheet. Sizing has logical forms too: inline-size and block-size replace width and height relative to flow.

Note: reach for logical properties by default. Even in a purely LTR project they cost nothing extra, and they leave the door open to internationalization without a rewrite.

Architecture

CSS scales badly by accident and well by convention. Two conventions dominate. BEM (Block, Element, Modifier) is a naming scheme that keeps selectors flat and self-documenting: a block is a component (.card), an element is a part of it (.card__title), and a modifier is a variant (.card--featured). Flat, single-class selectors mean low, uniform specificity and no cascade surprises.

ITCSS (Inverted Triangle CSS) organizes the stylesheet from most generic to most specific: settings and tools first, then generic resets, then elements, objects, components, and finally utilities. Ordering by increasing specificity means later rules naturally win without !important. Cascade layers (@layer) formalize this order in the language itself.

BEM naming (illustrative)

/* Block: the component root */
.card { }

/* Element: a part, double underscore */
.card__media { }
.card__title { }
.card__actions { }

/* Modifier: a variant, double hyphen */
.card--featured { }
.card__title--muted { }

/* Usage in markup:
   <article class="card card--featured">
     <h3 class="card__title">Featured</h3>
   </article> */
Result: every selector is a single class with specificity 0,1,0. Nothing needs to override anything by weight, so styles are predictable and safe to delete.

ITCSS layering (illustrative)

/* Order low-to-high specificity. With cascade layers: */
@layer settings, tools, generic, elements, objects, components, utilities;

@layer settings  { :root { --brand: #6366f1; } }   /* variables */
@layer generic   { *, *::before, *::after { box-sizing: border-box; } }
@layer elements  { a { color: var(--brand); } }     /* bare tags */
@layer objects   { .stack > * + * { margin-block-start: 1rem; } }
@layer components{ .card { border-radius: 12px; } } /* BEM blocks */
@layer utilities { .u-hidden { display: none; } }   /* single-purpose */
Result: utilities always win over components, components over objects, and so on — enforced by layer order, not selector specificity.

Utility classes vs component classes

Utility-first (e.g. .p-4 .flex .gap-2) composes styling in markup: fast to build, no naming, but verbose HTML and styling logic that lives in templates. Component classes (BEM) centralize a component's look behind one name: cleaner markup and a single place to change, but you must invent and maintain names. Most mature codebases blend the two — components for recurring, meaningful UI; utilities for one-off spacing and layout tweaks.

Example: a runnable BEM component with a modifier

Further reading

/* Modern features worth adopting next */
@layer reset, base, components, utilities;   /* cascade layers */

.form:has(input:invalid) .submit {           /* :has() parent selector */
  opacity: .5; pointer-events: none;
}

.gallery { display: grid; grid-template-columns: repeat(3, 1fr); }
.gallery__row { display: grid; grid-template-columns: subgrid; }  /* subgrid */

@media (prefers-reduced-motion: no-preference) {
  .reveal {                                  /* scroll-driven animation */
    animation: fade-in linear both;
    animation-timeline: view();
  }
}
Result: layers keep precedence explicit, :has() styles a parent from its children, subgrid aligns nested tracks to the outer grid, and scroll-driven animations run off scroll position with no JavaScript.

Next steps to deepen your CSS from here:

  • Cascade layers (@layer) — architect large stylesheets so precedence is intentional rather than accidental.
  • The :has() relational selector — style an element based on what it contains, enabling patterns that once required JavaScript.
  • Subgrid — let nested grids inherit their parent's track lines so cards and rows align perfectly.
  • Scroll-driven animations — drive keyframes from scroll or element visibility with animation-timeline, no scroll listeners.

Dashboard patterns

An Employee Dashboard is mostly a responsive shell wrapped around a grid of cards. The reliable base is a single-column, mobile-first layout: write the small-screen styles with no media query, then add min-width queries that progressively introduce columns and a sidebar as space allows. The card grid itself can reflow without any breakpoint using grid-template-columns: repeat(auto-fit, minmax(MIN, 1fr)) — the browser fits as many MIN-wide columns as the container allows and stretches them to share the row.

Two forces set the column count. auto-fit collapses empty tracks so present cards fill the width; the minmax() floor decides when a new column can appear. Layer explicit min-width breakpoints on top only where you need a deliberate structural change — for example flipping a stacked header-plus-sidebar shell into a two-track layout with the sidebar beside the content instead of above it.

Example: an auto-fit employee-card grid

Note: with auto-fit the column count is derived from available width, so one rule spans phone to widescreen. Raise the minmax() floor (say 220px) to force fewer, wider cards; lower it for a denser wall. No JavaScript and no resize listeners are involved.

Card interactions

Dashboard cards feel responsive when they lift on interaction. The pattern is a resting box-shadow plus a transform that raises the card on :hover and :focus-within, wired through a short transition. Pairing hover with :focus-within matters: keyboard users who tab into a card get the same affordance mouse users get on hover, so the feedback is not mouse-only.

Keep the motion cheap and considerate. Animate only transform and box-shadow (or opacity) so the work stays on the compositor, hint the browser with will-change where a heavier effect warrants it, and silence non-essential motion under prefers-reduced-motion: reduce. A flip or reveal built from transform is really the same idea taken one step further — two faces rotated in 3D space.

Example: an elevation hover/focus card

Note: pairing :hover with :focus-within gives keyboard users the same cue as mouse users, and :focus-visible keeps a clear ring without flashing it on every mouse click. Always provide a reduced-motion path — here the elevation stays but the travel is removed.

Note: the flip animates only transform, so it composites without triggering layout or paint. Add will-change: transform to .inner only if you measure jank — over-promoting many cards wastes GPU memory. The reduced-motion branch swaps the spin for a plain opacity cross-fade.

Collapsible sections

Profile details, notes, and settings groups all want to expand and collapse. The classic trick animated max-height between 0 and a guessed large value, but that guess is fragile — too small clips content, too large makes the easing time wrong because the transition covers empty space. The modern technique animates grid-template-rows from 0fr to 1fr: the row track resolves to the content's real height, so the reveal is smooth at any content length with no magic number.

The native <details>/<summary> element gives you an accessible disclosure for free — keyboard operable and announced by screen readers as expandable — and you can style the summary and animate its chevron. For real UI prefer a native <details> or a scripted button with aria-expanded; a bare checkbox toggle is not announced as a disclosure to assistive tech.

Values & units

Every length in CSS resolves to pixels eventually, but how it resolves is what matters. Absolute units like px are fixed; relative units resolve against something else. em resolves against the element's own font-size (and compounds through nesting), while rem resolves against the root font-size — so rem stays stable no matter how deep the element sits. Percentages resolve against a property-specific reference: width: 50% against the containing block, line-height: 150% against the font size.

Font-relative units include ch (the advance of a "0" glyph, ideal for measure), ex and cap (x-height and cap-height). Viewport units vw/vh/vmin/vmax track the window; the newer dvh/svh/lvh handle the collapsing mobile toolbar (dynamic, small, and large viewport heights). The grid-only fr unit distributes leftover space after fixed tracks are laid out. Transforms and transitions add deg/turn angles and s/ms times.

Typography & text

The font shorthand packs several properties into one declaration, but the order is strict: optional style/variant/weight/stretch, then the mandatory font-size (with an optional /line-height), then the mandatory font-family last. Omit size or family and the whole declaration is invalid. Self-hosted fonts arrive via @font-face; pairing it with font-display: swap shows fallback text immediately and swaps in the web font when it loads, avoiding invisible text.

Fine control comes from letter-spacing, word-spacing, text-transform, and text-decoration (now with text-decoration-thickness and text-underline-offset). white-space governs wrapping and collapsing; overflow-wrap and word-break break long strings. Truncate one line with text-overflow: ellipsis or several with -webkit-line-clamp. Variable fonts expose weight and other axes through font-variation-settings, and font-feature-settings/font-variant toggle ligatures and numeric figures.

Colors & gradients

A color can be named, hex (including the 8-digit #rrggbbaa form with alpha), or a function: rgb(), hsl(), and hwb() all accept the modern space-separated syntax with an optional / alpha. Perceptual spaces oklch() and oklab() describe color the way the eye reads it, so equal steps in lightness look equally spaced — invaluable for generating tint/shade ramps that do not muddy in the middle.

color-mix() blends two colors in a chosen space, which lets one design token spawn hover and active variants. currentColor refers to the element's own color, so borders and SVG fills track the text. Note the difference between opacity (fades the whole element and its children) and an alpha channel (fades one specific color). Gradients come in linear, radial, and conic flavors, each with a repeating- variant, and accept multiple stops with explicit positions to make hard-edged bands.

Sizing & object-fit

A computed size can be clamped from both ends: min-width/min-height set a floor the box can never drop below, and max-width/max-height set a ceiling. The intrinsic keywords let content decide the size — min-content shrinks to the widest unbreakable word, max-content grows to fit everything on one line, and fit-content() behaves like max-content but caps at a limit you supply. aspect-ratio locks a width-to-height proportion without the old padding-top hack.

Replaced elements — images, video, <canvas> — size differently from normal blocks: they have intrinsic dimensions. When you force one to fill a box, object-fit decides how the pixels map onto that box: cover fills and crops, contain fits and letterboxes, fill stretches, none keeps native size. object-position then nudges which part stays visible.

Note: setting aspect-ratio (or width/height attributes on an <img>) reserves the box before the image loads, which prevents the layout shift that hurts Cumulative Layout Shift scores.

Backgrounds, borders & shadows

A single element can paint several background layers at once: give background-image a comma-separated list and match it with per-layer background-size, -position, -repeat, and -attachment. The first layer sits on top. background-clip decides how far a background extends — border-box, padding-box, content-box, or the special text value that paints the background only through the glyph shapes for gradient text.

Borders offer per-side control, elliptical border-radius (the / separates horizontal and vertical radii), and border-image for sliced graphics. box-shadow accepts a comma list too, stacking multiple shadows and inset shadows for depth and inner highlights. outline draws outside the border and — crucially — takes no space in layout, which is exactly why it is the right tool for focus rings.

Overflow & scrolling

When content is larger than its box, overflow decides what happens: visible (the default) lets it spill, hidden clips it, scroll always shows scrollbars, auto shows them only when needed, and clip clips without creating a scroll container. Any value other than visible also establishes a block formatting context, which contains floats and stops margin collapse. overflow-x and overflow-y set the two axes independently.

Scroll behavior is tunable in CSS alone. scroll-behavior: smooth animates jumps to in-page anchors. Scroll snapping pins children to positions: set scroll-snap-type on the scroll container and scroll-snap-align on each child. overscroll-behavior stops scroll from chaining to the page behind a modal, and scrollbar-gutter: stable reserves the scrollbar's space so content does not shift when it appears.

Advanced selectors & pseudo-elements

The intro selectors section covered type, class, and simple pseudo-classes. This one adds the relational and structural power tools. :has() selects an element based on what it contains or what follows it — the long-awaited "parent selector" — enabling whole interactions with zero JavaScript. :not() negates, and the An+B microsyntax inside :nth-child()/:nth-last-child() targets arithmetic patterns like every third item.

State pseudo-classes react to the page and to forms: :target matches the element whose id is in the URL hash, :checked reflects a control's state, and :valid/:invalid/:required/:disabled style form fields by validity. Highlight and structural pseudo-elements — ::selection, ::first-line, ::first-letter, and ::backdrop for a top-layer <dialog> — style parts of an element that no markup can reach.

Native nesting & @scope

CSS now nests rules natively, no preprocessor required. Inside a rule you write child rules that reference the parent with &, and you can nest @media/@container blocks too. A nested selector that starts with a combinator or & is unambiguous; a nested type selector must be written with an explicit & (e.g. & span) because a bare span would be misread. Under the hood, nesting relates to :is(), which affects specificity: a nested selector takes the specificity of the most specific selector in its parent list.

@scope limits where rules apply using a scope root and an optional lower bound — the "donut" pattern @scope (.card) to (.content) styles the card chrome but stops at nested content. Both features complement the BEM naming and @layer cascade layers covered in the architecture section: nesting improves authoring readability, scope constrains reach, and layers order precedence.

Note: native nesting is Baseline in current browsers. Keep nesting shallow — two or three levels — because deep nesting produces long, brittle selectors and quietly raises specificity, the same problems it caused in Sass.

Feature & advanced media queries

@supports tests whether the browser understands a declaration or selector before you use it, so you can ship a fallback and layer an enhancement on top. It composes with not, and, or, and selector() to gate on selector support (e.g. @supports selector(:has(*))). This is feature detection in CSS, the counterpart to progressive enhancement.

Media queries read far more than width. Interaction features — hover, any-hover, pointer, any-pointer — distinguish a fine mouse from a coarse touch screen so you can size tap targets appropriately. Preference features like prefers-contrast, forced-colors, and inverted-colors adapt to accessibility settings, while orientation and the modern range syntax (@media (width >= 600px) and (400px <= width <= 900px)) replace the verbose min-/max- pairs.

Filters, blending, clipping & masking

The filter property runs graphics operations on an element: blur(), brightness(), contrast(), grayscale(), drop-shadow(), and more, chained left-to-right in one declaration. backdrop-filter applies the same functions to whatever sits behind a semi-transparent element, which is how frosted-glass panels are made — it needs a translucent background and a backdrop to sample.

mix-blend-mode composites an element with the layers below it, and background-blend-mode blends an element's own stacked backgrounds. clip-path hides everything outside a shape — circle(), inset(), or an arbitrary polygon() — while mask/mask-image use an image or gradient's alpha to reveal and hide parts of a box, perfect for edge fades and cut-outs.

The concepts below round out the page. Each is a confirmed gap deferred here to keep the document within its size budget; the one-line summary and the related section give you an anchor to explore each one, and every item is Baseline in current browsers unless noted.

  • @property & typed custom properties — Register a custom property with @property, declaring its syntax (e.g. <angle>, <color>, <number>, <length>, <percentage>), inherits flag, and initial-value. Registration is what makes a variable animatable: a plain --var jumps between values, but a registered <angle> or <color> interpolates smoothly, letting you transition conic-gradient angles, gradient color stops, and number counters that raw custom properties cannot. Invalid values fall back to the declared initial. Relates to Variables & calc() (extends untyped custom properties) and Transforms & animations (unlocks interpolation).
  • Forms & UI styling — accent-color themes native checkboxes, radios, ranges, and progress bars in one declaration; caret-color tints the text cursor. appearance: none strips native chrome so you can rebuild a checkbox with a :checked ::before checkmark or restyle a <select> with a custom chevron. Validation styling should prefer :user-valid/:user-invalid over :valid/:invalid so errors appear only after interaction, not on first paint; also style ::placeholder and disabled/read-only states while keeping an accessible focus indicator. Relates to Advanced selectors & pseudo-elements (validity pseudo-classes) and Colors & gradients (theming tokens).
  • Lists, counters & generated content — Style markers with list-style-type/-position/-image and the ::marker pseudo-element. Build custom numbering with counter-reset and counter-increment, rendering it through content: counter(name); the counters() function joins nested counters for outline numbering like 1.1, 1.2. attr() pulls an attribute value into generated content (e.g. a data-* prefix). Remember that generated content is decorative and not reliably exposed to assistive tech, so never put essential meaning there. Relates to Selectors (the ::before/::after and counter() intro) and Advanced selectors & pseudo-elements (structural pseudo-elements).
  • Print & paged media — An @media print block hides navigation and interactive chrome and switches to ink-friendly black-on-white. The @page rule sets page margin/size and supports named selectors like :first and :left/:right. Fragmentation control — break-inside: avoid, break-before/break-after, plus widows and orphans — keeps cards, table rows, and paragraphs from splitting awkwardly across pages. You can even expand link targets for print with a::after { content: " (" attr(href) ")" }.
