Web Design Cheatsheet

HTML, CSS & JavaScript — every concept from the Web Design course at a glance: syntax, a tiny example, and the result. Condensed for quick reference. Use the filter box to jump to any concept. Each card is a reminder, not a tutorial — for depth, see the full guides.

HTML Foundations

Document skeleton <!DOCTYPE html>

Every page opens with the DOCTYPE declaration, then <html> wrapping a hidden <head> (metadata) and a visible <body>.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>My Page</title>
  </head>
  <body>Hello</body>
</html>
OutputBrowser tab reads "My Page"; the word "Hello" shows in the page body.

Element anatomy & nesting <p>…</p>

An element is an opening tag + content + closing tag; nest freely but close tags in the reverse order you opened them.

<div>
  <p>Hello <b><i>there</i></b></p>
</div>
<!-- WRONG: overlapping -->
<b><i>x</b></i>
OutputCorrect nesting renders "there" bold-italic; overlapping tags produce unpredictable results.

Block vs inline <div> vs <span>

A block element takes a full-width line of its own; an inline element flows within a line of text.

<div>I am a block (own line)</div>
<div>Next block, new line</div>
<p>My name is <span>Ada</span> Lovelace.</p>
OutputThe two divs stack vertically; "Ada" stays inline within the sentence.

Headings, paragraphs & comments <h1>

Six heading levels (h1–h6) chosen by outline importance, not size; comments <!-- --> never render.

<h1>Main title</h1>
<h2>A section</h2>
<!-- this note is invisible -->
<p>A paragraph of text.</p>
Outputh1 large, h2 smaller, then a paragraph; the comment is hidden.

Void (self-closing) tags <br> <hr> <img>

Void elements carry no content, so they have no closing tag: br, hr, img, input, meta.

Line one<br>Line two
<hr>
<input placeholder="Type here">
OutputText splits across two lines, a divider rule, then a live input field.

Attributes & global attributes id class title

Attributes are quoted name="value" pairs in the opening tag; id is unique, class repeats, and both help CSS/JS target elements.

<h2 id="main-title">Title</h2>
<p class="note" title="Hover me!">Styled by class.</p>
<p class="note">Same class, same look.</p>
OutputBoth .note paragraphs share styling; hovering the first shows a tooltip.

Links & the anchor tag <a href>

The href is the destination: absolute URL, relative path, #in-page anchor, mailto: or tel:; add rel="noopener" with target="_blank".

<a href="about.html">About (relative)</a>
<a href="https://mozilla.org" target="_blank"
   rel="noopener">MDN (new tab)</a>
<a href="#bottom">Jump down</a>
<a href="mailto:hi@example.com">Email</a>
OutputFour links: same-tab page, new-tab site, in-page scroll, and an email draft.

Images <img src alt>

Void element; src is required, alt describes it for screen readers and broken-image fallback. Set width/height to prevent layout shift.

<img src="cat.jpg" alt="A grumpy cat"
     width="200" height="130">
<!-- responsive scale -->
<img src="cat.jpg" alt="Cat" style="width:100%;height:auto">
OutputImage shows at 200×130; if the file is missing, the alt text appears instead.

Lists: ul, ol, dl <li> <dt> <dd>

<ul> bullets, <ol> numbers (both hold <li>), <dl> pairs terms with descriptions; nest a list inside an <li>.

<ol>
  <li>Boil water
    <ul><li>Add tea</li></ul>
  </li>
  <li>Pour</li>
</ol>
<dl><dt>HTML</dt><dd>Page structure</dd></dl>
OutputNumbered steps with a bulleted sub-item, plus a term/description pair.

Tables <table> <tr> <th> <td>

Rows (<tr>) hold header (<th>) and data (<td>) cells; group with <thead>/<tbody>. For data only, not layout.

<table>
  <thead>
    <tr><th>Name</th><th>Score</th></tr>
  </thead>
  <tbody>
    <tr><td>Ada</td><td>98</td></tr>
  </tbody>
</table>
OutputGrid with a bold header row (Name, Score) above one data row (Ada, 98).

Table spans & caption colspan rowspan

<caption> names the table; colspan merges cells across columns, rowspan down rows.

<table>
  <caption>Room schedule</caption>
  <tr><td rowspan="2">Keynote</td><td>09:00</td></tr>
  <tr><td>10:00</td></tr>
  <tr><td colspan="2">Coffee break</td></tr>
</table>
Output"Keynote" fills two rows tall; "Coffee break" stretches across both columns.

HTML5 semantic elements <header> <main> <footer>

Meaningful landmarks that look like divs but describe their role — better SEO and accessibility for free.

<header>Logo</header>
<nav>Links</nav>
<main>
  <article>A self-contained post</article>
  <aside>Related links</aside>
</main>
<footer>© 2026</footer>
OutputSame visual stack as divs, but screen readers can jump between landmarks.

The <head>: metadata & meta tags

Invisible setup: charset for encoding, the viewport meta (the #1 responsive must-have), description for search snippets, and <title> for the tab.

<head>
  <meta charset="utf-8">
  <meta name="viewport"
        content="width=device-width, initial-scale=1">
  <meta name="description" content="A short page summary.">
  <title>My Page</title>
</head>
OutputCorrect encoding, mobile-friendly scaling, a search-result blurb, and a tab title.

HTML entities &lt; &amp; &copy;

Reserved characters need escapes: &lt;→<, &gt;→>, &amp;→&. Also handy: &copy;→© and &nbsp; (non-breaking space).

<p>Use &lt;p&gt; to write a paragraph.</p>
<p>Tom &amp; Jerry &copy; 2026</p>
<p>10&nbsp;km (never wraps)</p>
OutputShows literal "<p>", "Tom & Jerry © 2026", and "10 km" kept on one line.

More semantics <section> <figure>

<section> groups a themed block (usually with a heading); <figure> wraps media with a <figcaption> label.

<section>
  <h2>Gallery</h2>
  <figure>
    <img src="cat.jpg" alt="A cat">
    <figcaption>Caption</figcaption>
  </figure>
</section>
OutputA titled section containing an image with a caption beneath it.

Styling with CSS

Three ways to add CSS <link>

Inline styles one element, internal styles one page, external styles the whole site — prefer external.

<p style="color: teal;">inline</p>         <!-- one element -->

<head>
  <style> p { color: teal; } </style>    <!-- one page -->
  <link rel="stylesheet" href="styles.css"> <!-- whole site -->
</head>
OutputSame look three ways; external is cached and reusable.

Rule syntax & comments sel { prop: val; }

A selector picks elements, then property:value pairs in braces, each ended by a semicolon. Comments are block-only.

/* selector { property: value; } — no line comments in CSS */
h1 {
  color: navy;
  font-family: system-ui, sans-serif;
}
OutputAll <h1> become navy in a system sans-serif.

Selectors, specificity & the cascade #id .class tag

An id beats a class beats a tag; equal specificity means the later rule wins. Inline and !important outrank all.

p     { color: gray; }    /* 0-0-1  tag   */
.note { color: orange; }  /* 0-1-0  class beats tag */
#lead { color: red; }     /* 1-0-0  id beats class  */
Output#lead is red, .note orange, other <p> gray — highest specificity wins.

Colors color / background-color

Write a color as a name, hex, rgb(), or hsl(); add an alpha 0–1 with / for transparency.

.a { background-color: tomato; }              /* named  */
.b { background-color: #0b63ce; }             /* hex    */
.c { background-color: rgb(22 163 74); }      /* 0–255  */
.d { background-color: hsl(280 70% 45% / 0.5); } /* hue + alpha */
OutputFour fills; the last is 50% transparent so the page shows through.

Box model & box-sizing box-sizing: border-box

Every element is content + padding + border + margin. border-box folds padding/border into width so sizes stay predictable.

* { box-sizing: border-box; }  /* set globally */
div {
  width: 200px; padding: 20px; border: 4px solid #0b63ce;
}
/* content-box: real width 248px; border-box: stays 200px */
OutputWith border-box the box occupies exactly 200px wide.

Text properties text-align line-height

Control alignment, decoration, case, spacing, and line height of running text.

.title {
  text-align: center;
  text-transform: uppercase;   /* display only, HTML unchanged */
  letter-spacing: 3px;
  text-decoration: underline;
}
.lead { line-height: 1.7; text-align: justify; }
OutputCentered, spaced, underlined UPPERCASE heading; roomy justified body.

Fonts font-family stack & shorthand

Give a fallback stack ending in a generic family; set size, weight (400/700), and italic style — or use the font shorthand.

body { font-family: "Fira Sans", system-ui, sans-serif; }
p    { font-size: 20px; font-weight: 700; font-style: italic; }
/* shorthand: style weight size/line-height family (size+family required) */
h2   { font: italic 700 20px/1.5 Georgia, serif; }
OutputBold italic 20px text; browser falls back down the stack.

Link states & pseudo-classes :hover :focus

Style interactive states in LoVe-HAte order (link, visited, hover, active); keep :focus visible for keyboards.

a:link    { color: #0b63ce; }   /* unvisited */
a:visited { color: #7c3aed; }   /* already visited */
a:hover   { background: #0b63ce; color: white; }
a:active  { background: #073d80; }
a:focus   { outline: 3px solid orange; }
OutputLink inverts on hover; a Tab focus ring stays visible.

Structural pseudo-classes :nth-child() :not()

Match elements by position among siblings, or exclude a set with :not().

li:first-child      { font-weight: bold; }
li:nth-child(even)  { background: #f1f5f9; }  /* zebra stripes */
li:nth-child(3)     { color: tomato; }
button:not(.primary){ opacity: 0.7; }
OutputEven rows shaded, first item bold, non-primary buttons dimmed.

Display block / inline / inline-block / none

block takes a new line and is sizable; inline ignores width/height; inline-block is in-flow yet sizable; none removes the box.

.chip { display: inline-block; width: 80px; padding: 8px 0; }
.gone { display: none; }        /* removed, no space kept */
/* visibility: hidden keeps the space */
OutputChips sit in a row but honor their fixed 80px width; .gone disappears.

Position relative / absolute / fixed / sticky

Offsets (top/left…) only apply when position isn't static; absolute anchors to the nearest positioned ancestor.

.card  { position: relative; }   /* reference box */
.badge {
  position: absolute;            /* measured from .card */
  top: -10px; right: -10px; z-index: 2;
}
.header { position: sticky; top: 0; }  /* pins after scroll */
OutputBadge sits on the card's corner; header sticks to the top when scrolling.

Float, clear & clearfix float clear

Float pulls a box aside so text wraps around it; contain all-floated children with a clearfix (or display: flow-root).

.thumb { float: left; width: 70px; margin: 0 12px 6px 0; }
.wrap::after {        /* clearfix: stop parent collapsing */
  content: ""; display: block; clear: both;
}
OutputText wraps around the floated thumbnail; container keeps its height.

Useful properties overflow max-width opacity

Everyday utilities: cap width and center, scroll overflow, fade, change the cursor, and outline without shifting layout.

.column { max-width: 320px; margin: 0 auto; } /* centered column */
.scroll { height: 70px; overflow: auto; }      /* scrollbar if needed */
.cta    { cursor: pointer; }
.cta:hover { opacity: 0.8; }                   /* fade on hover */
OutputCapped centered column; inner box scrolls; button fades and shows a pointer.

Rounding, shadow & transitions border-radius transition

Round corners, drop a soft shadow, and animate property changes smoothly over a duration.

.card {
  border-radius: 10px;
  box-shadow: 0 10px 20px rgba(0,0,0,.15);
  transition: transform .18s ease, box-shadow .18s ease;
}
.card:hover { transform: translateY(-4px); }
OutputRounded card lifts and deepens its shadow smoothly on hover.

Combinators A B   A>B   A+B   A~B

Express relationships: descendant (space), direct child (>), adjacent sibling (+), general sibling (~).

article p   { color: #333; }      /* descendant: any p inside */
article > p { font-weight: bold; } /* child: direct only */
h3 + p      { color: #0b63ce; }    /* adjacent: first p after h3 */
h3 ~ p      { font-style: italic; }/* general: every p after h3 */
OutputDescendant p gray, direct-child p bold; + and ~ then walk siblings after h3.

Pseudo-elements ::before ::after ::first-letter

Style sub-parts or inject generated content; ::before/::after need a content value.

.story::first-letter { font-size: 3em; float: left; }  /* drop cap */
.story::first-line   { font-variant: small-caps; }
.tag::before { content: "\2605 "; color: gold; }        /* a star */
.tag::after  { content: " (read this)"; color: #888; }
OutputBig drop-cap first letter, small-caps first line, star + suffix around .tag.

Responsive design @media + viewport

Add the viewport meta, use relative units (%, rem, vw), and override styles at breakpoints — mobile-first.

<meta name="viewport" content="width=device-width, initial-scale=1">
.col { width: 100%; }                 /* mobile default */
@media (min-width: 640px) {           /* wider screens */
  .col { width: 50%; }
}
OutputFull-width columns on phones become two-up above 640px.

Flexbox display: flex

Lay children along one axis; justify-content = main axis, align-items = cross axis; flex: 1 shares leftover space.

.bar {
  display: flex;
  align-items: center;          /* cross axis */
  justify-content: space-between;
  gap: 16px;
  flex-wrap: wrap;
}
.bar .fill { flex: 1; }         /* grow to fill */
OutputItems in a centered row, spaced apart, wrapping when space runs out.

CSS Grid grid-template-columns fr

Lay out rows and columns at once; the fr unit splits free space and gap adds gutters.

.page {
  display: grid;
  grid-template-columns: 1fr 3fr;  /* sidebar + main */
  gap: 8px;
}
header, footer { grid-column: 1 / -1; }  /* span all columns */
OutputNarrow sidebar beside a 3× main column; header/footer span full width.

Responsive grid repeat(auto-fit, minmax())

Make columns wrap automatically: fit as many tracks as fit, each at least a minimum width, filling the row.

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(120px, 1fr));
  gap: 12px;
}
/* auto-fit collapses empty tracks; auto-fill keeps them */
OutputCards reflow from many columns to one as the viewport narrows — no media query.

Loading web fonts <link> / @import / display=swap

Pull a hosted font via a <link> in the head or @import atop your CSS, then name it in font-family. display=swap shows fallback text immediately (FOUT), avoiding invisible text (FOIT).

<!-- in <head>, before your stylesheet -->
<link rel="stylesheet"
  href="https://fonts.googleapis.com/css2?family=PT+Sans&display=swap">
/* or, at the very top of your CSS */
@import url("https://fonts.googleapis.com/css2?family=PT+Sans&display=swap");
body { font-family: "PT Sans", sans-serif; }
OutputText renders in a fallback first, then swaps to PT Sans once it loads — no blank flash.

JavaScript Fundamentals

Script tag & console <script defer>

Load JS with defer so the HTML parses first; console.log() prints to the dev-tools console (F12).

<script src="app.js" defer></script>
<!-- runs after HTML is parsed -->
console.log('Script is running');
console.log(2 + 2);
OutputScript is running
4

Variables const / let

Use const by default, let only to reassign. Both are block-scoped; avoid var (function-scoped, hoisted surprisingly).

const name = 'Web Design';  // fixed
let score = 0;              // can change
score += 15;
// name = 'x';  // TypeError: reassign const
Outputscore is 15; reassigning name throws.

Data types & typeof

Primitives: string, number, boolean, null, undefined. typeof reveals the type — note typeof null is the historic quirk 'object'.

typeof 'text';    // 'string'
typeof 42;        // 'number'
typeof true;      // 'boolean'
let x;            // declared only
typeof x;         // 'undefined'
typeof null;      // 'object' (quirk)
Outputstring, number, boolean, undefined, object

Template literals `${x}`

Backtick strings interpolate expressions with ${...} and span multiple lines — cleaner than + concatenation.

const name = 'Ada', year = 1843;
console.log(`${name} coded in ${year}.`);
console.log(`Line one
Line two`);
OutputAda coded in 1843.
Line one / Line two

Objects obj.key / obj['key']

Key-value collections. Read with dot notation, or bracket notation when the key is dynamic or not a valid identifier.

const user = {
  first: 'Ada',
  langs: ['JS', 'Python'],
};
user.first;        // 'Ada'  (dot)
user['first'];     // 'Ada'  (bracket)
user.langs[1];     // 'Python'
OutputAda, Ada, Python

Arrays push() / pop()

Ordered lists, zero-indexed. .length counts items; push appends, pop removes from the end.

const scores = [10, 20, 30];
scores[0];          // 10
scores.length;      // 3
scores.push(40);    // [10,20,30,40]
scores.pop();       // 40  -> back to length 3
Output10, 3, then push/pop mutate the array.

Operators === && %

Arithmetic + - * / % **, assignment shorthands (+= ++), and logical && || !. Prefer === over == (no type coercion).

let val = 10;
val += 5;             // 15
val % 7;              // 1
10 === '10';          // false (type differs)
10 == '10';           // true  (coerces — avoid)
true && false;        // false
true || false;        // true
Output15, 1, false, true, false, true

Functions function f(){}

Reusable named block. Parameters are placeholders, arguments the real values; return hands a value back and ends the function.

function adder(a, b) {
  return a + b;   // ends the function
}
adder(5, 10);     // 15
adder(117, 80);   // 197
Output15, then 197

Arrow functions =>

Shorter function expression. A single-expression body has an implicit return (no braces); one param can skip the parentheses.

const double = n => n * 2;        // implicit return
const add = (a, b) => a + b;
const greet = (who = 'friend') => `Hi, ${who}!`;
double(4);   // 8
add(2, 3);   // 5
Output8, 5, "Hi, friend!"

Conditionals if / else & ternary

Only the first matching branch runs. The ternary cond ? x : y is a compact inline alternative that produces a value.

function grade(score) {
  if (score >= 90) return 'A';
  else if (score >= 70) return 'B';
  else return 'Fail';
}
grade(72);                       // 'B'
const age = 20;
const canVote = age >= 18 ? 'yes' : 'no';
Outputgrade(72) -> "B"; ternary picks yes/no.

Switch switch/case

Branch on one value against many cases. Stacked cases fall through to share a block; default is the catch-all.

function describe(day) {
  switch (day) {
    case 'Sat':
    case 'Sun':
      return 'weekend';   // grouped cases
    default:
      return 'weekday';
  }
}
describe('Sun');   // 'weekend'
Outputweekend

Loops for / while / do…while

for packs start, condition, update in one line. while may run zero times; do…while always runs at least once.

for (let i = 0; i < 3; i++) console.log(i);  // 0 1 2

let n = 0;
while (n < 3) { n++; }        // 0 or more times

let m = 5;
do { console.log(m); } while (m < 3);  // runs once
Output0 1 2; do…while prints 5 once.

Iterating arrays for…of / forEach

for…of gives each value; forEach calls a function with value and index. (for…in yields keys — use it for objects.)

const fruit = ['apple', 'kiwi', 'mango'];
for (const item of fruit) console.log(item);
fruit.forEach((value, index) => console.log(index, value));
Outputapple, kiwi, mango
0 apple, 1 kiwi, 2 mango

Reference vs value types

Top beginner trap: const b = arr copies a reference, so mutating b also changes arr. Primitives copy by value and never share.

const scores = [10, 20, 30];
const b = scores;   // copies the reference, not contents
b[1] = 999;
scores[1];          // 999 — changed through b!

let x = 5, y = x;   // primitive: copied by value
y = 99;
x;                  // still 5
Outputscores[1] is 999; x stays 5.

The DOM

The DOM tree document

The browser parses your HTML into a live tree of nodes; document is the root you script from.

document.title;              // page title
document.body;               // the <body> element
const ul = document.querySelector('ul');
ul.parentElement;            // node above
ul.children;                 // element children
ul.firstElementChild;        // first <li>
OutputNavigate up (parent) and down (children) from any node.

Selecting elements querySelector()

Grab nodes by id or CSS selector; querySelectorAll returns a NodeList you loop with forEach.

document.getElementById('fav');      // one, no # prefix
document.querySelector('.item');     // first match
const all = document.querySelectorAll('.item'); // all
all.forEach((li, i) => console.log(i, li.textContent));
OutputNo match: querySelector returns null; querySelectorAll an empty list.

Content: text vs markup textContent

textContent sets plain text (safe); innerHTML parses a string as HTML into real elements.

el.textContent = 'Changed with JS';   // literal text
box.innerHTML  = '<strong>Bold</strong>'; // parsed markup
// Never put user input in innerHTML → XSS risk.
OutputtextContent shows the tags as text; innerHTML renders real bold text.

Attributes get/setAttribute()

Read and write any attribute; common ones (id, src, href, value) are also direct properties.

el.setAttribute('data-role', 'label');
el.getAttribute('data-role');   // "label"
el.removeAttribute('disabled');
img.src = 'photo.png';          // property shortcut
OutputgetAttribute('data-role') → "label"

Styling: style & classList classList.toggle()

Set inline styles via element.style, but prefer flipping a CSS class with classList (add/remove/toggle/contains).

el.style.color = 'crimson';        // inline style
el.classList.add('active');
el.classList.remove('active');
el.classList.toggle('highlight');  // on if off, off if on
el.classList.contains('active');   // true/false
Outputtoggle returns true when the class was just added.

Reading form input input.value

Every field's current text lives on its .value property; read it in an event or on submit.

const name = document.getElementById('name');
console.log(name.value);           // current text
const agree = document.getElementById('tos');
console.log(agree.checked);        // checkbox → boolean
OutputLogs whatever the user has typed into the field.

Event listeners addEventListener()

Register a handler for an event type; the handler receives an event object. Multiple listeners stack (unlike onclick).

btn.addEventListener('click', (event) => {
  event.target.textContent = 'Clicked!'; // element that fired
});
// { once: true } auto-removes after first call
btn.addEventListener('click', fn, { once: true });
Outputevent.target is the node the event came from.

Forms & preventDefault e.preventDefault()

A form submit reloads the page by default; cancel it with preventDefault() and handle the data in JS.

form.addEventListener('submit', (e) => {
  e.preventDefault();              // stop the reload
  console.log('Sending:', field.value);
});
OutputForm no longer reloads; you process field.value yourself.

Common events click / input / keydown

The events you reach for most: pointer, live typing, committed changes, submit, and keys.

el.addEventListener('click', fn);   // pointer/tap
el.addEventListener('input', fn);   // fires on every keystroke
el.addEventListener('change', fn);  // after value commits/blur
form.addEventListener('submit', fn);
document.addEventListener('keydown', (e) => console.log(e.key));
Outputinput fires per keystroke; change only once done.

DOMContentLoaded DOMContentLoaded

Fires once the HTML is parsed and the tree exists; run setup here so a <head> script still finds its elements.

document.addEventListener('DOMContentLoaded', () => {
  const out = document.getElementById('out'); // safe now
  out.textContent = 'Ready';
});
OutputSelectors find their targets instead of returning null.

Creating & inserting nodes createElement()

Make a detached node, fill it, then appendChild to attach it. node.remove() deletes it from the tree.

const list = document.getElementById('list');
fruits.forEach((name) => {
  const li = document.createElement('li'); // 1. make
  li.textContent = name;                   // 2. fill
  list.appendChild(li);                    // 3. attach
});
list.firstElementChild.remove();           // delete a node
OutputA new <li> per fruit appears; first one is removed.

Animation requestAnimationFrame()

Schedule a callback just before the next repaint (~60fps); re-call it each frame for smooth motion, cancel to stop.

let x = 0, id;
function frame(timestamp) {
  x += 2;
  ball.style.left = x + 'px';
  id = requestAnimationFrame(frame); // queue next frame
}
id = requestAnimationFrame(frame);   // start loop
cancelAnimationFrame(id);            // stop it
OutputBall glides across; pauses automatically in hidden tabs.

Bubbling & delegation event.target

Events bubble from the target up through its ancestors; put one listener on a parent and branch on event.target to handle many children.

list.addEventListener('click', (e) => {
  const li = e.target.closest('li'); // which child?
  if (!li) return;                   // clicked the gap
  li.classList.toggle('done');
});
// one listener covers current AND future <li>s
OutputClicking any <li> toggles it — no per-item listeners.

Removing listeners removeEventListener()

Detach a handler by passing the same function reference used to add it; an inline arrow can never be removed because it has no name to match.

function onClick() { console.log('hi'); }
btn.addEventListener('click', onClick);
btn.removeEventListener('click', onClick); // works

// ✗ different function each time → cannot remove
btn.addEventListener('click', () => {});
OutputOnly the named-function listener detaches; the arrow one stays forever.

Moving & reordering nodes insertBefore()

A node lives in exactly one place, so re-appending an existing node moves it (no copy). Use insertBefore / prepend to reorder.

list.appendChild(existingLi);        // moves it to the end
list.prepend(existingLi);            // moves it to the front
list.insertBefore(newLi, refLi);     // put newLi before refLi
first.after(second);                 // sibling insert
OutputThe node vanishes from its old spot and reappears at the new one.

Clearfix a floated multi-column layout ::after{clear:both}

Arrow functions =>

Lexical this

Arrows inherit this from the enclosing scope, so never use one as an object method that needs the object.

const obj = {
  n: 10,

  good() { return this.n; }

map / filter / reduce

Transform, keep-matching, and fold an array into one value — the workhorse array methods.

const nums = [1, 2, 3, 4, 5, 6];

every / some / find / includes

Test all, test any, get the first match, or check membership — each returns a single answer.

const scores = [82, 91, 78, 95];
scores.every(s => s >= 50);   // true  (all pass; true on [])
scores.some(s => s > 90);     // true  (at least one)
scores.find(s => s < 80);     // 78    (first match, else undefined)
scores.includes(91);         // true  (===  membership)

Advanced JavaScript

Custom map()

A higher-order function: loop the array, call the callback (value, index, array) per item, collect results in a new array. Never mutates the source.

function myMap(arr, cb) {
  const out = [];
  for (let i = 0; i < arr.length; i++) {
    out.push(cb(arr[i], i, arr));   // one call per item
  }
  return out;
}
myMap([1, 2, 3], (n) => n * 2);
Output[2, 4, 6] — same length, source untouched. Forget to return in the callback and you get [undefined, ...].

Minimal promise then/resolve

A tiny state machine: pending settles once to fulfilled or rejected, then queued .then() handlers run. A guard blocks any later resolve/reject.

class MyPromise {
  constructor(executor) {
    this.handlers = [];
    const settle = (state) => (value) => {
      if (this.state) return;        // settle only once
      this.state = state; this.value = value;
      this.handlers.forEach((h) => h(value));
    };
    executor(settle("fulfilled"), settle("rejected"));
  }
  then(onOk) {
    if (this.state) onOk(this.value);
    else this.handlers.push(onOk);
    return this;
  }
}
new MyPromise((res) => setTimeout(() => res("done"), 100))
  .then((v) => console.log(v));
Output"done" — logged once the promise resolves. Returning this is simplified chaining; real chains return a new promise per .then().

Deep clone (recursive)

Spread {...obj} and Object.assign are shallow — nested objects stay shared. Recurse to rebuild every branch; return primitives and null as-is.

function deepClone(v) {
  if (v === null || typeof v !== "object") return v; // null check FIRST
  if (Array.isArray(v)) return v.map(deepClone);
  const o = {};
  for (const k in v) if (Object.hasOwn(v, k)) o[k] = deepClone(v[k]);
  return o;
}
OutputFully independent copy; editing a nested key on the clone never touches the original. Crashes with a RangeError on circular refs.

Deep clone built-ins structuredClone()

Modern browsers ship structuredClone(obj) — handles nesting, cycles, Date, Map. The old JSON round-trip is a hack that silently drops data.

const deep = structuredClone(obj);          // preferred, handles cycles

// JSON round-trip caveat: drops functions, undefined,
// Symbols; turns Date into a string; throws on cycles.
const lossy = JSON.parse(JSON.stringify(obj));
OutputstructuredClone: faithful copy. JSON hack: { fn, d: Date } becomes { d: "2026-..." } — function gone, date now a string.

Debounce debounce(fn, delay)

Delays a function until a burst of rapid calls stops. Each call clears the previous timer via a closed-over id. Ideal for search-as-you-type, resize, scroll, autosave.

function debounce(fn, delay) {
  let id;                              // remembered via closure
  return function (...args) {
    clearTimeout(id);                  // cancel pending run
    id = setTimeout(() => fn.apply(this, args), delay);
  };
}
const onSearch = debounce((q) => console.log("search:", q), 500);
input.addEventListener("input", (e) => onSearch(e.target.value));
OutputFires once, 500ms after the last keystroke. Create it once — rebuilding it per event means nothing ever cancels.

Throttle vs debounce

Debounce's cousin: throttle runs the function at most once per interval during a burst; debounce waits for the burst to stop. Use throttle for scroll/mousemove, debounce for search input.

function throttle(fn, interval) {
  let last = 0;
  return function (...args) {
    const now = Date.now();
    if (now - last >= interval) {       // enough time passed?
      last = now;
      fn.apply(this, args);             // run at most once/interval
    }
  };
}
window.addEventListener("scroll", throttle(onScroll, 200));
OutputThrottle: steady calls every 200ms while scrolling. Debounce: silence until scrolling pauses, then one call.

Promise .catch() / .finally()

A rejected promise skips .then and lands in .catch(onErr); .finally(fn) runs on either outcome for cleanup. .catch is just .then(null, onErr).

fetch("/api")
  .then((res) => res.json())          // runs on fulfill
  .catch((err) => console.error(err))  // runs on any reject
  .finally(() => hideSpinner());       // runs either way

// reject example
new Promise((res, rej) => rej(new Error("boom")))
  .catch((e) => console.log(e.message)); // "boom"
OutputOn reject: .then is skipped, .catch logs the error, then .finally still runs. An uncaught rejection surfaces as an "unhandled rejection" warning.
