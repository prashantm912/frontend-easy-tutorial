Angular Cheatsheet

Angular 17–22 — every core concept at a glance: syntax, a tiny example, and the output. Condensed from the Angular guide and the Angular Core Deep Dive. Use the filter box to jump to any concept. Each card is a reminder, not a tutorial — for depth, see the full guides.

Introduction

What Angular is @Component

A component-based framework: you build UI from components, each of which renders as its own custom HTML element extending the browser's tag set.

@Component({
  selector: 'app-root',
  standalone: true,
  template: '<h1>Hello</h1>',
})
export class AppComponent {}
Output<app-root> renders in the DOM as your own custom element.

Custom HTML elements

The page's <body> is nearly empty — one custom tag (the root component). Angular injects all visible content into it; the selector is the tag name.

<!-- index.html -->
<body>
  <app-root></app-root>
</body>
OutputTitle, links, and markup all appear inside <app-root>, not in index.html.

Model vs View {{ }}

The model is the data on the component class (TS); the view is the template (HTML). Interpolation binds them by evaluating a JS expression against the instance.

export class AppComponent {
  data = { title: 'Angular Core Deep Dive' };
}
// template: <h1>{{ data.title }}</h1>
Output<h1>Angular Core Deep Dive</h1>

A component = three files

Each component pairs a @Component-decorated class (model), an HTML template (view), and scoped CSS — wired via templateUrl/styleUrl.

@Component({
  selector: 'app-root',
  standalone: true,
  templateUrl: './app.component.html',
  styleUrl: './app.component.css',
})
export class AppComponent {}
OutputClass, template, and scoped styles combine to render one custom element.

Why Angular — key features

Property/event binding, automatic change detection (view auto-syncs to the model), built-in XSS-escaping, scoped styles, DI, and a full CLI/tooling ecosystem.

<input #box [value]="data.title"
       (keyup)="onKeyUp(box.value)">
<h1>{{ data.title }}</h1>
OutputTyping updates the <h1> automatically; any <script> in data is escaped, not run.

Modern vs classic styles

Signals + new control flow (@if/@for) are the modern style; decorators + structural directives (*ngIf) are classic. Both are supported and coexist in real code.

<!-- modern -->
@if (show) { <p>Hi</p> }

<!-- classic -->
<p *ngIf="show">Hi</p>
OutputBoth render <p>Hi</p> when show is true; codebases often mix the two.

CLI & tooling ecosystem

Angular is more than a framework — it ships a CLI. ng new scaffolds a project; ng serve runs a dev server with hot reload.

ng new my-app        # scaffold a new project
cd my-app
ng serve             # dev server + hot reload
OutputApp builds and serves at http://localhost:4200, reloading on every save.

Components, Core Directives & Pipes

First component @Component

A component = a decorated class (the model) + a template (the view) + scoped styles; selector is its custom tag.

@Component({
  selector: 'course-card',
  standalone: true,
  templateUrl: './course-card.component.html',
  styleUrl: './course-card.component.css'
})
export class CourseCardComponent {}
Output<course-card></course-card> renders the component's template wherever the tag appears.

Interpolation {{ expr }}

One-way binding from model to view: the expression is evaluated against the component instance and can reach into objects.

<h2>{{ course.description }}</h2>
<p>{{ data.title }}</p>
OutputRenders the class property values as text; expressions must be side-effect free.

Pass data down @Input()

Marks a child property as a bindable input; the parent feeds it via square-bracket property binding. Pass a whole typed object over many primitives.

export class CourseCardComponent {
  @Input({ required: true }) course!: Course;
}
<course-card [course]="coreCourse"></course-card>
OutputEach card shows its own course; required: true is a compile error if omitted.

Emit events up @Output() + EventEmitter

A child announces intent by emitting a custom event; the parent listens with the parentheses binding and reads the payload via $event.

@Output() courseSelected = new EventEmitter<Course>();
onClick() { this.courseSelected.emit(this.course); }
<course-card (courseSelected)="onSelected($event)" />
OutputParent's handler receives the emitted Course. Custom events do NOT bubble — only @Output exposes them.

Conditional render @if / @else

Built-in control flow (Angular 17+): renders a block when truthy, with @else if / @else fallbacks. No import needed.

@if (course.id === 1) {
  <div>Course with id one</div>
} @else if (course.iconUrl) {
  <img [src]="course.iconUrl">
} @else {
  <div>No image available</div>
}
OutputExactly one branch renders; the element is removed from the DOM (not just hidden).

Loop @for — track, @empty & vars

Iterates an iterable; track is mandatory (identity for DOM reuse). Exposes $index $count $first $last $even $odd; @empty handles zero items.

@for (course of courses; track course.id; let i = $index) {
  <course-card [course]="course" [class.first]="$first" />
} @empty {
  <h2>No courses found</h2>
}
OutputOne card per item, indexed from 0; empty array shows the fallback. Omitting track is a compile error.

@for tracking functions track

Track by a stable unique key (item.id) for best diffing; use a function for computed identity, or $index for primitive lists that never reorder.

@for (course of courses; track course.id) { … }
@for (course of courses; track trackCourse($index, course)) { … }
@for (tag of tags; track $index) { <span>{{ tag }}</span> }
OutputStable keys yield minimal DOM operations and preserve focus/scroll state.

Legacy loop *ngFor

The legacy structural directive that @for replaced; aliases locals with let (index first last even odd) and takes an optional trackBy. Needs CommonModule.

<course-card
  *ngFor="let course of courses; let i = index; trackBy: trackCourse"
  [course]="course" [index]="i"></course-card>
OutputSame list rendering; unlike @for, forgetting trackBy compiles fine (silent perf trap).

Legacy *ngIf + else template

Conditional structural directive; the else branch points at a named <ng-template>. Guard nullables with the Elvis operator ?.

<p *ngIf="user; else anon">{{ user?.name }}</p>
<ng-template #anon><p>Not signed in</p></ng-template>
OutputShows the name when user exists, else the fallback template.

Legacy *ngSwitch

Multi-branch legacy directive: [ngSwitch] on the parent, *ngSwitchCase / *ngSwitchDefault on children. Case must live on a child, not the host.

<div [ngSwitch]="course.category">
  <div *ngSwitchCase="'BEGINNER'">Beginner</div>
  <div *ngSwitchCase="'ADVANCED'">Advanced</div>
  <div *ngSwitchDefault>All Levels</div>
</div>
OutputRenders the child whose case strictly equals the expression, else the default.

Blocks vs directives ng generate control-flow

@for/@if/@switch are compiled, need no imports, narrow types, and make tracking mandatory; the directives are slower and must be imported. Auto-migrate the codebase.

ng generate @angular/core:control-flow
OutputRewrites *ngIf/*ngFor/*ngSwitch to @if/@for/@switch across the project (~95% automatic).

Multi-branch @switch / @case / @default

Modern control flow that renders one branch by strict equality (===) — no fall-through, no break, no import.

@switch (course.category) {
  @case ('BEGINNER')     { <div>Beginner</div> }
  @case ('INTERMEDIATE') { <div>Intermediate</div> }
  @default               { <div>All Levels</div> }
}
OutputOnly the matching case renders; without a @default an unmatched value shows nothing.

Conditional classes [ngClass]

Toggles state classes from a string, array, or (most useful) object map of class → boolean. Keep static classes on the plain class attribute.

<div class="course-card"
     [ngClass]="{ beginner: isBeginner, highlighted: isActive }">
OutputAdds each class whose value is true. For one class, prefer native [class.beginner]="isBeginner".

Dynamic styles [ngStyle]

Applies several inline styles at once from an object of CSS property → value. Use only for data-driven properties; static styling belongs in a stylesheet.

<div [ngStyle]="{
  'background-image': 'url(' + course.iconUrl + ')',
  'font-size.px': 14
}">
OutputSets the listed inline styles; unit suffixes like .px / .% let you bind a plain number.

Group without an element <ng-container>

A logical wrapper that renders no DOM of its own — host a structural directive or group siblings without polluting markup with an extra <div>.

<ng-container *ngFor="let u of users">
  <li *ngIf="u.active">{{ u.name }}</li>
</ng-container>
OutputOnly the inner <li> elements reach the DOM; solves "two structural directives on one element".

Built-in pipes value | pipe:arg

Transform data for display with |; pass args after colons. Catalog: date, uppercase/lowercase/titlecase, number/percent/currency, json, slice, keyvalue, async.

{{ startDate | date:'MMMM d, yyyy' }}
{{ title | uppercase }}   {{ title | titlecase }}
{{ price | number:'3.3-5' }}   {{ price | currency:'EUR' }}
{{ rate | percent }}   {{ (courses | slice:0:2).length }} shown
<pre>{{ course | json }}</pre>
@for (p of course | keyvalue; track p.key) { {{ p.key }}: {{ p.value }} }
{{ user$ | async }}
Output"January 1, 2000"; "ANGULAR" / "Angular"; "009.990"; "€9.99"; "67%"; "2 shown"; JSON dump; object entries; unwrapped observable.

Capture with as alias

Bind the evaluated expression to a local so you can reuse it — the common async-unwrap pattern: subscribe once, then read fields off the alias.

@if (user$ | async; as user) {
  <p>{{ user.name }} — {{ user.email }}</p>
}

<p *ngIf="course as c">{{ c.title }}</p>
OutputBlock renders only when truthy; user/c hold the unwrapped value for the whole block — one subscription, no repeated | async.

Auto-subscribe | async

Impure pipe: subscribes to an Observable/Promise in the template, renders the latest value, and unsubscribes on destroy — no manual cleanup, no leaks.

<!-- two async = two subscriptions/requests -->
<p>{{ user$ | async }}</p>
<p>{{ user$ | async }}</p>

<!-- unwrap once with as -->
@if (user$ | async; as user) { {{ user.name }} }
OutputRe-renders on each emission; piping the same stream twice makes two subscriptions — unwrap once with as to share one.

Template Querying

Grab a child @ViewChild()

Pull a rendered child component, directive, or element up into the class so code can drive it imperatively. A component locator returns the instance.

import { Component, ViewChild } from '@angular/core';

@Component({ /* imports: [CourseCardComponent] */ })
export class AppComponent {
  @ViewChild(CourseCardComponent) card!: CourseCardComponent;

  onClick() {
    console.log(this.card.course.description); // read a prop
    this.card.highlight();                     // call a method
  }
}
OutputLogs the child's course description; runs highlight() on the live instance.

Locators & read:

Locate by component class, by template-ref-variable name (string), or a plain element. A component returns its instance; read: ElementRef overrides that to hand back the host DOM node.

// by ref name → component instance
@ViewChild('cardOne') cardOne!: CourseCardComponent;

// plain element → ElementRef (.nativeElement is the node)
@ViewChild('container') box!: ElementRef;

// component, but read its host element instead
@ViewChild(CourseCardComponent, { read: ElementRef })
cardEl!: ElementRef;
Outputbox.nativeElement is the real <div>; cardEl.nativeElement is the <app-course-card> host.

Query timing static: true | false

Default static: false resolves the field in ngAfterViewInit; static: true resolves it in the first change-detection pass so it is ready in ngOnInit — only for non-conditional single elements.

@ViewChild('container', { static: true }) box!: ElementRef;

ngOnInit() {
  // available now because static:true
  console.log(this.box.nativeElement);
}
OutputLogs the <div> in ngOnInit. With static:false it would be undefined here.

Ready in ngAfterViewInit()

Query fields are undefined in the constructor and ngOnInit; Angular fills them once the view is built and calls AfterViewInit. Queries are local — never into a child's template, never up to the parent.

export class AppComponent implements AfterViewInit {
  @ViewChild('container') box!: ElementRef;

  constructor() { console.log(this.box); }   // undefined
  ngAfterViewInit() { console.log(this.box.nativeElement); } // ready
}
Outputctor: undefined — afterViewInit: <div class="courses">…</div>

Many matches @ViewChildren + QueryList

Returns every match as a QueryList (not an Array). Use first/last/length/map/forEach; subscribe to changes to react when @for adds or removes items.

@ViewChildren(CourseCardComponent)
cards!: QueryList<CourseCardComponent>;

ngAfterViewInit() {
  console.log(this.cards.first, this.cards.length);
  this.cards.forEach(c => console.log(c.course.id));
  this.cards.changes.subscribe(list =>
    console.log('changed → length', list.length));
}
OutputLogs first + count now; emits new length when the collection changes (not on initial fill).

QueryList to real Array toArray()

A QueryList is iterable but not an Array — filter/reduce are absent. Call toArray() first when you need genuine array methods.

ngAfterViewInit() {
  const arr = this.cards.toArray();          // CourseCardComponent[]
  const rest = arr.filter(c => c.course.id > 1);
  console.log('total:', arr.length, 'filtered:', rest.length);
}
Outputtotal: 10 filtered: 9

Signal queries viewChild() / viewChildren()

Reactive counterpart: functions return signals — no static flag, no AfterViewInit needed. viewChild.required() is non-nullable; viewChildren() gives a Signal<readonly T[]>. Derive with computed().

import { Component, viewChild, viewChildren, computed } from '@angular/core';

export class AppComponent {
  card = viewChild(CourseCardComponent);            // Signal<Cmp | undefined>
  box  = viewChild.required<ElementRef>('container'); // Signal<ElementRef>
  cards = viewChildren(CourseCardComponent);         // Signal<readonly Cmp[]>

  count = computed(() => this.cards().length);

  onClick() {
    this.card()?.highlight();          // read the signal, then use
    console.log(this.box().nativeElement, this.count());
  }
}
Outputcard() may be undefined; box() always resolved; count() recomputes as @for adds/removes items.

read: beyond ElementRef

Not just ElementRef — read picks which token to pull off the matched element: TemplateRef, ViewContainerRef, or a specific directive instance sharing the host.

// grab a template's blueprint
@ViewChild('tpl', { read: TemplateRef }) tpl!: TemplateRef<unknown>;

// get an insertion point to render into
@ViewChild('slot', { read: ViewContainerRef }) slot!: ViewContainerRef;

// one directive instance off a host with several
@ViewChild(MyTooltip, { read: MyHighlight }) hl!: MyHighlight;
Outputtpl.createEmbeddedView(...) / slot.createComponent(...) / hl is the co-located directive.

Content Projection

Single slot <ng-content>

A bare <ng-content> is a landing spot for whatever markup the parent writes between the component's tags.

@Component({
  selector: 'ui-card', standalone: true,
  template: `<div class="card"><ng-content></ng-content></div>`
})
export class CardComponent {}

// Usage:
<ui-card><h2>Hello</h2></ui-card>
OutputThe parent's <h2>Hello</h2> renders inside the card's .card div.

Multi-slot select=

Named slots route projected nodes by select (element, .class, or [attr]); a selector-less slot catches the rest.

<section class="panel">
  <header><ng-content select="[panel-title]"></ng-content></header>
  <div><ng-content></ng-content></div>              <!-- catch-all -->
  <footer><ng-content select="[panel-actions]"></ng-content></footer>
</section>

<ui-panel>
  <h3 panel-title>Title</h3>
  <p>Body text</p>
  <button panel-actions>OK</button>
</ui-panel>
OutputTitle → header, button → footer, the <p> → catch-all body. Each node lands in exactly one slot.

Query projected node @ContentChild

Gets a class-level handle on a single projected node (by template ref or type). Ready in ngAfterContentInit, not the constructor.

@Component({
  selector: 'ui-card', standalone: true,
  template: `<ng-content></ng-content>`
})
export class CardComponent implements AfterContentInit {
  @ContentChild(CourseImageComponent) image?: CourseImageComponent;
  // by ref: @ContentChild('cover', { read: ElementRef }) el!: ElementRef;

  ngAfterContentInit() { console.log(this.image); }
}
OutputLogs the projected CourseImageComponent instance; returns undefined (silently) if nothing matches.

Query a collection @ContentChildren

Returns a QueryList of all matching projected nodes; read it in ngAfterContentInit. Subscribe .changes for dynamic updates.

export class AccGroupComponent implements AfterContentInit {
  @ContentChildren(AccPanelComponent)
  panels!: QueryList<AccPanelComponent>;

  ngAfterContentInit() {
    console.log(this.panels.length);
    this.panels.first?.open();      // coordinate children
  }
}
Output3 — panel count is known and the first panel opens once content is initialised.

Signal queries contentChild()

Modern signal forms return a signal (or Signal<readonly T[]>) — read reactively, no lifecycle hook required.

export class CardComponent {
  image = contentChild(CourseImageComponent);       // Signal<T | undefined>
  panels = contentChildren(AccPanelComponent);       // Signal<readonly T[]>

  count = computed(() => this.panels().length);
}
Outputcount() tracks the projected panels reactively — no ngAfterContentInit needed.

View children vs content children

@ViewChild queries the component's own template (ready at ngAfterViewInit); @ContentChild queries projected content (ready at ngAfterContentInit). Neither crosses into the other.

@ViewChild('ownEl')  own!: ElementRef;      // from THIS template
@ContentChild('proj') proj!: ElementRef;     // projected via <ng-content>
// content query inspects what's passed between the tags,
// even if the template has NO <ng-content> to render it.
OutputContent queries populate at ngAfterContentInit, view queries at ngAfterViewInit; both are undefined in the constructor and ngOnInit.

@ContentChildren({ descendants: true })

By default a content query finds only direct projected children; pass { descendants: true } to include matches nested deeper in the projected tree.

// Only top-level projected <app-tab>:
@ContentChildren(Tab) shallow!: QueryList<Tab>;

// Also Tabs wrapped inside other projected elements:
@ContentChildren(Tab, { descendants: true })
deep!: QueryList<Tab>;
OutputIf a Tab sits inside a nested <div>, shallow misses it but deep includes it.

Projection gotchas

Each projected node lands in exactly one slot; select matches only top-level nodes by a static compile-time selector — you can't switch slot at runtime.

<!-- Duplicate select= → only the LAST slot renders it -->
<ng-content select="[x]"></ng-content>
<ng-content select="[x]"></ng-content>  <!-- wins -->

<!-- Content in a false @if STILL instantiates
     (its hooks run) but isn't attached to the DOM -->
@if (open) { <ng-content></ng-content> }
OutputNo duplication across slots, no runtime slot switching, and projected content is always constructed even when its @if is false.

Templates (ng-template)

Declaring a template <ng-template #ref>

Defines an inert fragment that is parsed but NOT rendered until something instantiates it; the #ref handle lets you pass it around.

<ng-template #blankImage>
  <div class="warn">No image available</div>
</ng-template>

<!-- referenced via an else branch -->
<img *ngIf="course.iconUrl; else blankImage" [src]="course.iconUrl">
OutputPlaceholder stays a ghost slot until iconUrl is falsy; remove the ref and it simply never reaches the DOM (no error).

Rendering it *ngTemplateOutlet

Stamps a TemplateRef on demand; host it on <ng-container> so no wrapper element is added.

<ng-template #blankImage>
  <div class="warn">No image available</div>
</ng-template>

<ng-container *ngTemplateOutlet="blankImage"></ng-container>
OutputRenders the fragment in place. Forget the leading * and nothing is stamped out.

Context + let- bindings

A context object gives each instance private variables; let-x reads $implicit, let-x="key" reads a named property.

<ng-template #greet let-name let-role="role">
  <p>Hello {{ name }} — {{ role }}</p>
</ng-template>

<ng-container *ngTemplateOutlet="greet;
  context: { $implicit: 'Ada', role: 'admin' }"></ng-container>
OutputHello Ada — admin. A missing/mis-spelled key just yields undefined — no error.

Reusing one template

The same TemplateRef with different contexts yields independent instances.

<ng-template #badge let-text>
  <span class="badge">{{ text }}</span>
</ng-template>

<ng-container *ngTemplateOutlet="badge; context: { $implicit: 'New' }"></ng-container>
<ng-container *ngTemplateOutlet="badge; context: { $implicit: 'Sale' }"></ng-container>
OutputTwo badges: New and Sale, each with its own scope.

TemplateRef as an @Input()

Pass a template into a component to customise its rendering — the component owns the loop, the caller controls each row's markup.

@Component({
  selector: 'app-list',
  template: `
    @for (item of items; track item) {
      <ng-container [ngTemplateOutlet]="rowTpl"
        [ngTemplateOutletContext]="{ $implicit: item }"></ng-container>
    }`,
})
export class ListComponent {
  @Input() items: unknown[] = [];
  @Input() rowTpl!: TemplateRef<{ $implicit: unknown }>;
}
OutputConsumer: <app-list [items]="users" [rowTpl]="row"> with <ng-template #row let-u>{{u.name}}</ng-template> — inverts control of appearance.

Grabbing a projected template @ContentChild(TemplateRef)

Idiomatic alternative to an explicit @Input(): the component reads a caller's <ng-template> from its projected content and stamps it — consumers write ergonomic markup inside the tags while the component owns instantiation.

@Component({
  selector: 'app-list',
  template: `
    @for (item of items; track item) {
      <ng-container [ngTemplateOutlet]="tpl"
        [ngTemplateOutletContext]="{ $implicit: item }"></ng-container>
    }`,
})
export class ListComponent {
  @Input() items: unknown[] = [];
  @ContentChild(TemplateRef) tpl!: TemplateRef<{ $implicit: unknown }>;
}
OutputConsumer: <app-list [items]="users"><ng-template let-u>…</ng-template></app-list>. Reads in ngAfterContentInit; matches nothing → undefined, not an error.

Directives In Depth

Attribute directive @Directive({ selector: '[appX]' })

Attaches behaviour to a host element without changing DOM structure; the bracketed attribute selector matches any element carrying that attribute. Inject ElementRef to reach the native node.

import { Directive, ElementRef, inject } from '@angular/core';

@Directive({ selector: '[appHighlight]', standalone: true })
export class HighlightDirective {
  private el = inject(ElementRef) as ElementRef<HTMLElement>;
  constructor() { this.el.nativeElement.style.outline = '2px solid red'; }
}
// <course-card appHighlight>  → one instance per matching element
OutputEvery element with the appHighlight attribute gets a red outline; dropping the attribute never instantiates the directive.

Prefer Renderer2 over nativeElement

Mutating nativeElement directly breaks under SSR/non-browser platforms; Renderer2 is the safe, platform-agnostic way to touch the DOM.

@Directive({ selector: '[appPulse]', standalone: true })
export class PulseDirective {
  private el = inject(ElementRef);
  private r  = inject(Renderer2);
  @HostListener('click') on() {
    this.r.addClass(this.el.nativeElement, 'pulsing');
    this.r.setAttribute(this.el.nativeElement, 'aria-busy', 'true');
  }
}
OutputClick adds the pulsing class and aria-busy="true" — safely, even during server-side rendering.

@HostBinding @HostBinding('class.x')

Declaratively writes a host class/style/property/attribute from a field or getter — Angular keeps it in sync with change detection, no imperative DOM poking.

@Directive({ selector: '[appHighlight]', standalone: true })
export class HighlightDirective {
  @Input('appHighlight') on = false;            // alias == selector

  @HostBinding('class.highlighted') get cls() { return this.on; }
  @HostBinding('style.border')     get bd()  { return this.on ? '2px solid red' : null; }
  @HostBinding('attr.data-on')     get flag(){ return String(this.on); }
}
OutputWhen [appHighlight]="true": <course-card class="highlighted" style="border:2px solid red" data-on="true">.

DOM property vs HTML attribute attr.

A bare name binds the live JS property; the attr. prefix writes the static HTML attribute. Use attr. for ARIA and anything with no matching property.

@HostBinding('value')        v = 'live';   // .value property (what user sees now)
@HostBinding('attr.value')   a = 'init';   // value="init" HTML attribute
@HostBinding('attr.aria-pressed') p = true; // no DOM property named 'aria-pressed' → attr. only
// note: colSpan IS a real property (camelCase) — @HostBinding('colSpan') would work too
// @HostBinding('doesNotExist') → runtime error: property does not exist
OutputThe markup value attribute holds only the initial string; .value tracks live edits. Binding an unknown property throws.

@HostListener @HostListener('event', ['$event'])

Runs a method whenever the named native event fires on the host; pass ['$event'] to receive the event object. Any DOM event works.

@Directive({ selector: '[appHighlight]', standalone: true })
export class HighlightDirective {
  @HostBinding('class.highlighted') on = false;

  @HostListener('mouseover', ['$event']) enter(e: MouseEvent) { this.on = true;  }
  @HostListener('mouseleave')            leave()             { this.on = false; }
}
OutputHovering adds .highlighted, leaving removes it. Use mouseleave, not mouseout — mouseout bubbles from children and flickers.

exportAs exportAs: 'name'

Names the directive instance so a template reference variable can grab its public API; query the same instance from the class with @ViewChild (ready in ngAfterViewInit).

@Directive({ selector: '[appHighlight]', exportAs: 'appHighlight', standalone: true })
export class HighlightDirective { toggle() { /* ... */ } }
<course-card appHighlight #hl="appHighlight" (dblclick)="hl.toggle()"></course-card>
OutputEach double-click calls the directive's toggle(). The exportAs string is chosen independently of the selector; a mismatched #hl="..." errors "no directive with exportAs set to ...".

Structural * desugaring *ngIf

The leading * is sugar: Angular wraps the host in an implicit <ng-template> and rebinds the directive as a property binding on it. Two structural directives can't share one element.

<!-- what you write -->
<div *ngIf="course as c">{{ c.title }}</div>

<!-- what Angular runs -->
<ng-template [ngIf]="course" let-c="ngIf">
  <div>{{ c.title }}</div>
</ng-template>
OutputBoth render identically. <div *ngIf *ngFor> is a compile error — wrap one in <ng-container>.

Custom structural directive TemplateRef + ViewContainerRef

Inject the wrapped TemplateRef (the stamp) and a ViewContainerRef (the slot). An input setter named exactly like the selector drives createEmbeddedView() (show) / clear() (hide).

import { Directive, Input, TemplateRef, ViewContainerRef } from '@angular/core';

@Directive({ selector: '[appUnless]', standalone: true })
export class UnlessDirective {
  private visible = false;
  constructor(private tpl: TemplateRef<any>, private vcr: ViewContainerRef) {}

  @Input() set appUnless(cond: boolean) {          // setter name == selector
    if (!cond && !this.visible) { this.vcr.createEmbeddedView(this.tpl); this.visible = true; }
    else if (cond && this.visible) { this.vcr.clear(); this.visible = false; }
  }
}
Output<p *appUnless="user.isAdmin"> shows for non-admins only (inverse of *ngIf). The visible flag skips redundant re-renders.

Directives emit @Output events

A directive is an event source too: expose an @Output() EventEmitter and emit() from a @HostListener, so the host element gains a custom event.

@Directive({ selector: '[appHover]', standalone: true })
export class HoverDirective {
  @Output() hovered = new EventEmitter<boolean>();

  @HostListener('mouseenter') on()  { this.hovered.emit(true);  }
  @HostListener('mouseleave') off() { this.hovered.emit(false); }
}
// <course-card appHover (hovered)="active = $event">
OutputThe host now fires a (hovered) event carrying a boolean — consumed exactly like a component's own output.

Query a directive with @ViewChild + read

Grab a directive instance from the class via @ViewChild(Type) (ready in ngAfterViewInit); add { read: Dir } to pick one specific directive off a host wearing several.

@ViewChild(HighlightDirective) hl!: HighlightDirective;

// disambiguate: read the directive, not the component, on one host
@ViewChild(CourseCard, { read: HighlightDirective })
hlOnCard!: HighlightDirective;

ngAfterViewInit() { this.hl.toggle(); }   // instance is ready here
Outputhl holds the directive instance so you can call its public methods; read resolves which token to return when a host has both a component and directives.

Context for createEmbeddedView

Pass a context object as the 2nd arg; $implicit becomes the default let-x and named keys bind via let-i="key" — this is how a structural directive exposes ngFor-style locals.

this.vcr.createEmbeddedView(this.tpl, {
  $implicit: item,   // → let-item
  index: i,          // → let-i="index"
});

  {{ i }}: {{ item }}
</p>
Output$implicit feeds the bare let item; index is reached with let i = index — same pattern the built-in *ngFor uses.

View Encapsulation

Emulated scoping _ngcontent

Default mode: the compiler stamps a per-component attribute on every element and rewrites each CSS selector to require it, so styles cannot leak out.

/* you write in course-card.component.css */
.course-card { box-shadow: 0 0 10px #0002; }

/* compiler rewrites it to: */
.course-card[_ngcontent-ng-c1] { box-shadow: 0 0 10px #0002; }
OutputOnly real <course-card> elements match; a plain .course-card div elsewhere is untouched.

Style the host :host

Targets the component's own element (compiles to [_nghost-id]); a bare tag-name selector fails because the host carries no content attribute.

:host {
  display: block;
  padding: 1rem;
}
/* conditional: host when it also has .selected */
:host(.selected) { border: 2px solid dodgerblue; }
OutputThe <course-card> wrapper becomes a padded block; a highlighted border appears only when it has class "selected".

Theme from an ancestor :host-context()

Styles this component based on a class on any ancestor (e.g. a theme wrapper), while the styled selector stays scoped to the component only.

/* when ANY ancestor has .salmon-theme, theme MY card only */
:host-context(.salmon-theme) .course-card {
  background-color: lightsalmon;
}
/* compiles to: .salmon-theme .course-card[_ngcontent-ng-c1] */
OutputThis component's card is themed; a plain .course-card sibling under the same ancestor is NOT.

Pierce the boundary ::ng-deep

Drops the content-attribute requirement so a rule reaches projected/descendant nodes; deprecated — always anchor it with :host so it stays contained.

/* reach content projected by a DIFFERENT component */
:host ::ng-deep .course-description {
  color: gray;
}
/* bare ::ng-deep .foo (no :host) leaks globally — avoid */
OutputProjected .course-description is styled. Prefer CSS custom properties or ::part instead — ::ng-deep is deprecated.

Modes ViewEncapsulation

Three modes: Emulated (default, attribute scoping), None (all CSS global), ShadowDom (native shadow root).

import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'app-badge',
  encapsulation: ViewEncapsulation.Emulated, // default, recommended
  // ViewEncapsulation.None      → styles become global
  // ViewEncapsulation.ShadowDom → real shadow root
})
export class BadgeComponent {}
OutputNone makes .badge a global rule that restyles any .badge element on the page; Emulated keeps it scoped. Native mode is deprecated — use ShadowDom.

Emulated vs ShadowDom isolation

Emulated is one-way (blocks leaking out, global styles still cascade in). ShadowDom is near two-way native isolation — outer non-inherited styles are blocked (inherited props and CSS vars still cross).

encapsulation: ViewEncapsulation.ShadowDom
/* <app-badge>
     #shadow-root
       <style>…</style>   ← lives INSIDE, blocks page cascade
       <span class="badge">…
OutputShadowDom silently blocks your global stylesheet/resets from reaching the component — switch modes deliberately.

Never hard-code generated ids

Pitfall: _ngcontent-ng-c1 / _nghost ids are build-time artifacts that change between builds and Angular versions — never target them in a selector or test; rely on encapsulation behavior instead.

/* BAD — brittle, breaks on the next build */
.course-card[_ngcontent-ng-c1] { color: red; }
[_nghost-ng-c3] { display: block; }

/* GOOD — let Angular scope it; you just write */
.course-card { color: red; }
:host { display: block; }
OutputThe hashed id suffix (ng-c1, ng-c3…) is regenerated per build, so any selector or test asserting on it silently breaks.

Enable HttpClient provideHttpClient()

// app.config.ts

  providers: [ provideHttpClient() ]

OutputHttpClient is now injectable app-wide.

GET with query params HttpParams

Fire a typed GET in ngOnInit. HttpParams is immutable, so each set() returns a new instance — chain them and pass under the params option.

private http = inject(HttpClient);

ngOnInit() {

    .set('pageSize', '10');

Subscribe to send the request .subscribe()

courses: Course[] = [];

ngOnInit() {
  this.http.get<Course[]>('/api/courses')
    .subscribe(courses => {
      this.courses = courses;   // feed the view
    });
}

Async pipe observable$ | async

Expose an Observable field ($ suffix) and let the template subscribe. It unwraps the latest value and auto-unsubscribes on destroy — no cleanup, no leaks.

courses$!: Observable<Course[]>;

  this.courses$ = this.http.get<Course[]>('/api/courses');

@if (courses$ | async; as courses) {

}
OutputList renders; subscription cleaned up automatically.

Custom service @Injectable

@Injectable({ providedIn: 'root' }) makes a class injectable and lets it receive its own deps. The root injector gives one shared singleton app-wide.

@Injectable({ providedIn: 'root' })
export class CoursesService {
  private http = inject(HttpClient);   // DI supplies it
  constructor() { console.log('created'); }
}
OutputInjected in many components → constructor logs exactly once.

Fetch & expose from a service loadCourses()

Move HTTP logic into an intent-revealing method returning Observable<Course[]>. The component depends on the service, not on HttpClient.

// courses.service.ts
loadCourses(): Observable<Course[]> {
  return this.http.get<Course[]>('/api/courses');
}

// app.component.ts

OutputSame 10 cards; HTTP detail hidden behind the service API.

Modify data http.put()

PUT takes (url, body, options) — the body is mandatory. Add headers via immutable HttpHeaders. It's a cold Observable, so subscribe() to actually save.

saveCourse(course: Course): Observable<any> {
  const headers = new HttpHeaders().set('X-Auth', 'token');
  return this.http.put(
    `/api/courses/${course.id}`,   // url with id
    course,                         // body (required)
    { headers });                   // options
}
// caller
this.svc.saveCourse(c).subscribe(() => console.log('saved'));
OutputPUT /api/courses/1 → 200 OK; new title persists on reload.

Cache with shareReplay(1)

Every subscribe on a raw http.get() re-fires the request. Pipe shareReplay(1) so one request runs and its result replays to all subscribers.

loadCourses(): Observable<Course[]> {
  return this.http.get<Course[]>('/api/courses')
    .pipe(shareReplay(1));   // one request, shared
}
// two subscribers now share a single GET
this.svc.loadCourses().subscribe();
this.svc.loadCourses().subscribe();
OutputWithout shareReplay: 2 GETs. With it: 1 GET, replayed.

Full verb set post/patch/delete

Beyond GET/PUT: post and patch take a body; delete and head don't. All return cold Observables — subscribe to send.

this.http.post<Course>('/api/courses', newCourse);
this.http.patch(`/api/courses/${id}`, { title });
this.http.delete(`/api/courses/${id}`);
this.http.head('/api/courses');
OutputNothing sent until each is subscribed.

Full response observe: 'response'

By default the body is unwrapped. Pass { observe: 'response' } to get the whole HttpResponse — status code and headers included.

this.http.get<Course[]>('/api/courses', { observe: 'response' })

    console.log(resp.headers.get('etag')); // header
    console.log(resp.body);                // the courses
  });

Dependency Injection In Depth

How DI works inject()

A class declares its dependencies (via inject() or constructor params); Angular's injector finds the matching provider, constructs the value once, and hands it over. No provider anywhere → NullInjectorError.

@Injectable({ providedIn: 'root' })
export class CoursesService {
  private http = inject(HttpClient);   // service's own dep
}

@Component({ selector: 'app-root', standalone: true })
export class AppComponent {
  private courses = inject(CoursesService);  // Angular supplies it
}
OutputInjector resolves & constructs CoursesService once. Remove its provider → NullInjectorError: No provider for CoursesService!

Providers & tokens { provide, useFactory, deps }

A provider is the recipe that produces a dependency; a token is the lookup key. A class doubles as its own token (it exists at runtime); an interface cannot.

// Fully explicit provider: token + recipe + its deps
providers: [
  {
    provide: CoursesService,               // token (the key)
    useFactory: (h: HttpClient) => new CoursesService(h),
    deps: [HttpClient],                    // resolved & passed to factory
  },
]
OutputFactory runs once on first request. A missing deps array → Can't resolve all parameters (Angular does not read the signature).

Simplified provider config useClass / useValue / useFactory / useExisting

Shorthands for the four provider recipes. [Service] expands to { provide: Service, useClass: Service }.

providers: [CoursesService]  // sugar for { provide, useClass }

// useClass  -> new instance of a class
// useValue  -> inject a ready-made object/primitive as-is
// useFactory-> run a fn (with deps) to compute the value
// useExisting-> ALIAS to an already-provided token (shared, no new copy)

let n = 0;  @Injectable() class Svc { id = ++n; }
providers: [Svc, { provide: API, useExisting: Svc }]  // one object
providers: [Svc, { provide: API, useClass:    Svc }]  // two objects
OutputuseExisting: inject(Svc).id === inject(API).id → true. useClass: false (a second instance).

Hierarchical injection providers: []

Injectors form a tree mirroring the component tree. A request searches the local injector, then climbs to parents up to root. A component-level provider mints a private instance for that subtree.

// Root singleton: inherited where no local provider exists
@Injectable({ providedIn: 'root' })
export class CoursesService { id = ++counter; }

// Own provider -> private instance PER rendered card
@Component({ selector: 'course-card', standalone: true,
            providers: [CoursesService] })
export class CourseCardComponent {
  private courses = inject(CoursesService);
}
OutputRoot uses #1; each card builds its own #2, #3…. Remove the card provider → every card falls back to root #1.

Tree-shakeable providers providedIn: 'root'

Configure the provider on the service itself so the bundler can drop it when nothing injects it. Registers an app-wide singleton and is the default choice for stateless services.

@Injectable({ providedIn: 'root' })   // tree-shakeable singleton
export class CoursesService {
  private http = inject(HttpClient);
}

// Also accepts the recipes; providedIn is required:
@Injectable({ providedIn: 'root',
  useFactory: (h: HttpClient) => new CoursesService(h), deps: [HttpClient] })
export class CoursesService {}
OutputUnused → code removed from the bundle. Injected → one lazy root instance. (Listing it in providers too creates a second, shadowing instance.)

InjectionToken for non-class values new InjectionToken<T>()

A type/interface has no runtime identity, so plain config objects need an explicit token. Give it a { providedIn, factory } to stay tree-shakeable.

export type AppConfig = { apiUrl: string };
export const APP_CONFIG: AppConfig = { apiUrl: 'http://localhost:9000' };

export const CONFIG_TOKEN = new InjectionToken<AppConfig>('CONFIG_TOKEN', {
  providedIn: 'root',
  factory: () => APP_CONFIG,          // tree-shakeable
});

private config = inject(CONFIG_TOKEN);  // must ask by token
// Non-tree-shakeable: providers: [{ provide: CONFIG_TOKEN, useValue: APP_CONFIG }]
OutputInjects the object. inject(AppConfig) (the type) fails — types vanish at runtime; inject the token value.

@Optional, @Self, @SkipSelf inject(X, { self })

Modifiers that steer resolution: tolerate a missing provider, forbid climbing, or force climbing past the local injector.

// null instead of throwing (must guard the null)
private log = inject(LoggerService, { optional: true });
// only this component's own provider (else error)
private local = inject(CoursesService, { self: true });
// ignore local; take a parent's instance
private parent = inject(CoursesService, { skipSelf: true });

// Classic decorator form:
// constructor(@Optional() log: LoggerService,
//   @Self() local: CoursesService, @SkipSelf() parent: CoursesService) {}
Outputself with no local provider → No provider error. optional with nothing to inject → null (no crash).

@Host decorator inject(X, { host: true })

Caps a directive's upward search at its host element's injector; the host component (or the directive) must provide it, or injection errors — it will not fall back to root.

@Directive({ selector: '[highlighted]', standalone: true })
export class HighlightedDirective {
  private courses = inject(CoursesService, { host: true });  // no climbing to root
}

@Component({ selector: 'course-card', standalone: true,
  imports: [HighlightedDirective],
  providers: [CoursesService],   // REQUIRED for host-scoped injection
})
export class CourseCardComponent {}
// constructor(@Host() private courses: CoursesService) {}
OutputWith host provider: gets the card's private instance. Remove it → NullInjectorError: No provider (found by @Host).

Injection context runInInjectionContext()

inject() only works inside an injection context: field initializers, the constructor, or a factory. Calling it later (e.g. in a click handler) throws; capture deps up front or pass an Injector.

export class CoursesComponent {
  private http = inject(HttpClient);       // OK: field initializer
  private injector = inject(Injector);
  constructor() { const r = inject(Router); }  // OK: constructor

  onClick() {
    const svc = inject(CoursesService);    // THROWS: outside context
    runInInjectionContext(this.injector, () => inject(CoursesService));  // OK
  }
}
OutputLate inject() → NG0203: inject() must be called from an injection context. Grab deps as fields, or wrap in runInInjectionContext.

Change Detection

Default change detection

Active out of the box: after any async event Angular re-checks the whole tree top-down, comparing each binding's current value to its previous one. Zone.js is what signals when to run.

@Component({
  selector: 'course-card',
  template: `<h2>{{ course.description }}</h2>
    <input #t (keyup)="onChange(t.value)">`
})
export class CourseCardComponent {
  @Input() course!: { description: string };
  onChange(v: string) {
    this.course.description = v; // mutation is fine
  }
}
OutputEvery keyup fires a pass; the <h2> updates live. Direct mutation works because each expression is re-read.

OnPush strategy ChangeDetectionStrategy.OnPush

Opt in for speed: put OnPush on the display child; it is checked only on an @Input reference change, an in-component event, an async-pipe emission, or a signal read change — otherwise its whole subtree is skipped.

@Component({
  selector: 'course-card',
  changeDetection: ChangeDetectionStrategy.OnPush,   // on the child
  template: `<h2>{{ course.description }}</h2>`
})
export class CourseCardComponent {
  @Input() course!: { description: string };
}

@Component({
  selector: 'app-root',   // parent owns the buttons + data
  template: `<course-card [course]="course"></course-card>`
})
export class AppComponent {
  course = { description: 'Angular Basics' };
  mutate()  { this.course.description = 'Edited'; }        // ignored
  replace() { this.course = { ...this.course,
                              description: 'Edited' }; }    // updates
}
Output"Mutate" leaves the OnPush card stale (same reference); "Replace" swaps the reference and re-renders. Immutable updates pair naturally with OnPush.

OnPush with Observables | async

Under OnPush, feed streams through the async pipe — each emission marks the component for check. A manual subscribe + field assignment is no trigger and leaves the view blank.

@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  imports: [AsyncPipe],
  template: `
    @if (courses$ | async; as courses) {
      @for (c of courses; track c.id) {
        <course-card [course]="c"></course-card>
      }
    }`
})
export class AppComponent implements OnInit {
  courses$!: Observable<Course[]>;
  constructor(private svc: CoursesService) {}
  ngOnInit() { this.courses$ = this.svc.loadCourses(); }
}
OutputList renders when the HTTP response lands and repaints on every emission; the pipe also unsubscribes on destroy.

The @Attribute decorator

Reads a static host attribute once at construction — never re-checked on later passes. Use for values fixed for the component's lifetime (a "kind"/"variant" string).

import { Attribute, Component } from '@angular/core';

@Component({
  selector: 'course-card',
  template: `<div class="card {{ type }}">...</div>`
})
export class CourseCardComponent {
  constructor(@Attribute('type') public type: string) {
    console.log('card type =', this.type); // once per instance
  }
}
// <course-card type="beginner"></course-card>
OutputLogs once per instance from the constructor; later clicks/timers never re-read it. Value is always a string (or null if absent).

Manual control markForCheck()

Inject ChangeDetectorRef; markForCheck() flags the component and its ancestors for the next pass — the fix after a manual subscribe under OnPush. Put per-pass logic in ngDoCheck.

constructor(private svc: CoursesService,
            private cd: ChangeDetectorRef) {}

ngOnInit() {
  this.svc.loadCourses().subscribe(list => {
    this.courses = list;
    this.loaded = true;
  });
}
ngDoCheck() {
  if (this.loaded) {
    this.cd.markForCheck();   // schedule next pass
    this.loaded = false;      // render first batch only
  }
}
OutputngDoCheck fires every pass; markForCheck() runs once so the initial list renders. Prefer | async over this.

detectChanges & detach/reattach

detectChanges() runs a pass on this component now, synchronously. detach() removes a subtree from automatic checking; reattach() restores it — pair with a timer to throttle a high-frequency stream.

constructor(private cd: ChangeDetectorRef) {}

ngOnInit() {
  this.cd.detach();                 // off automatic checking
  setInterval(() => {
    this.value = this.stream.now();
    this.cd.detectChanges();        // repaint once per second
  }, 1000);
}
// this.cd.reattach();  // hand control back to Angular
OutputA firehose stream repaints at most once per second instead of on every emission. Use all of this sparingly, as a last resort.

Lifecycle Hooks

Setup ngOnInit()

Runs once after inputs are set — the home for init logic; the constructor should only capture injected deps (inputs are still undefined there).

export class CourseCard implements OnInit {
  @Input() course: any;
  constructor() { console.log(this.course); } // undefined
  ngOnInit() { console.log(this.course); }    // set (sync inputs)
}
Outputconstructor: undefined  ·  ngOnInit: { title: 'Angular Core' }

Teardown ngOnDestroy()

Fires once just before Angular removes the component — release manual subscriptions here (often unneeded: the async pipe / takeUntilDestroyed() auto-clean).

export class CourseCard implements OnDestroy {
  private sub = interval(1000).subscribe();
  ngOnDestroy() { this.sub.unsubscribe(); } // free resources
}
OutputCalled when an @if wrapping the component turns false and removes it.

React to input changes ngOnChanges()

Runs before ngOnInit and again on every input change. SimpleChanges maps each input to previousValue, currentValue, firstChange.

ngOnChanges(changes: SimpleChanges) {
  const c = changes['course'];
  console.log(c.firstChange, c.previousValue, c.currentValue);
}
// Fires only on a NEW reference:
this.courses = [{ ...this.courses[0], title: 'x' }, ...rest];
Outputtrue, undefined, {…}  →  false, {old}, {new}. Mutating a property in place does NOT fire it.

Projected content ready ngAfterContentInit / …Checked

Content = markup a parent passes into <ng-content>. Init runs once (@ContentChild queries ready); Checked runs every cycle — keep it cheap.

ngAfterContentChecked() {
  this.course.category = 'ADVANCED';   // OK: not in projected content
  // this.course.iconUrl = '';         // THROWS if bound in <ng-content>
}
OutputPitfall: mutating a checked projected value throws ExpressionChangedAfterItHasBeenCheckedError (dev mode). No backend calls here.

View + children ready ngAfterViewInit / …Checked

View = the component's own template. Init runs once (@ViewChild populated); Checked runs every cycle — for DOM ops only, never data writes.

@ViewChild('list') list!: ElementRef;
ngAfterViewInit() { console.log(this.list); } // now defined
ngAfterViewChecked() {
  const el = this.list.nativeElement;
  el.scrollTop = el.scrollHeight;             // DOM ready: safe
}
OutputScrolls the freshly rendered list to the bottom. Writing a bound value here throws the changed-after-checked error.

Complete hook order

Fixed sequence per component. Init hooks run once; Checked/DoCheck repeat every change-detection cycle; destroy runs once on removal.

constructor()            // capture injected deps only
ngOnChanges(changes)     // inputs set/changed by REFERENCE
ngOnInit()               // once: init using inputs
ngDoCheck()              // every cycle: custom change detection
ngAfterContentInit()     // once: @ContentChild(ren) ready
ngAfterContentChecked()  // every cycle
ngAfterViewInit()        // once: @ViewChild(ren) ready
ngAfterViewChecked()     // every cycle: DOM ops
ngOnDestroy()            // once: cleanup
OutputMost components need only ngOnInit. Signal inputs (input()) avoid ngOnChanges entirely.

Custom change detection ngDoCheck()

Runs every cycle to catch mutations Angular's reference check misses (e.g. an in-place array edit); pair with OnPush + cd.markForCheck(). Keep it very cheap.

@Component({ changeDetection: ChangeDetectionStrategy.OnPush })
export class Cart implements DoCheck {
  @Input() items: Item[] = [];
  private len = 0;
  constructor(private cd: ChangeDetectorRef) {}
  ngDoCheck() {
    if (this.items.length !== this.len) {   // detect in-place push()
      this.len = this.items.length;
      this.cd.markForCheck();               // re-render this component
    }
  }
}
OutputFires on items.push(x) where ngOnChanges would stay silent (same array reference). Runs on EVERY cycle — do the minimum.

Async inputs are undefined in ngOnInit

Sync-bound inputs are ready in ngOnInit, but an input fed by an async/HTTP source is still undefined there — it only arrives later via ngOnChanges.

// Parent: <course-card [course]="course$ | async"></course-card>
export class CourseCard implements OnInit, OnChanges {
  @Input() course?: Course;
  ngOnInit() { console.log(this.course); }      // undefined (data not back yet)
  ngOnChanges(c: SimpleChanges) {
    console.log(c['course'].currentValue);      // arrives here when HTTP resolves
  }
}
OutputngOnInit: undefined  →  ngOnChanges: { title: 'Angular Core' }. React to the value in ngOnChanges, not ngOnInit.

NgModule metadata @NgModule

@NgModule({

@NgModule({

Custom pipe @Pipe + PipeTransform

A decorated class that transforms a value for display; transform's return replaces the expression in the view.

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({ name: 'filterByCategory' })  // standalone default v19+
export class FilterByCategoryPipe implements PipeTransform {
  transform(courses: Course[], category: string): Course[] {
    return (courses ?? []).filter(c => c.category === category);
  }
}
// template:  *ngFor="let c of (courses | filterByCategory:'BEGINNER')"
OutputValue left of | is the 1st param; each :-arg is the next. Only BEGINNER courses reach the loop.

@NgModule({

})
export class CoursesModule {}
OutputForget to register and the build fails with "no pipe found with name 'filterByCategory'".

Pure vs impure pipes pure: false

Pure (default) re-runs only on a new input reference; impure re-runs every change-detection cycle — the performance trade-off.

@Pipe({ name: 'filterByCategory' })          // PURE (default)
// re-runs only when the 'courses' reference changes

@Pipe({ name: 'filterByCategory', pure: false }) // IMPURE
// transform() runs on EVERY CD cycle — catches in-place mutation
OutputMutating the array in place leaves a pure pipe stale; pure:false fixes it but runs constantly. Prefer immutable updates + a pure pipe.

BrowserModule vs CommonModule

Root module imports BrowserModule once (browser render services; it re-exports CommonModule); feature modules import CommonModule for *ngIf/*ngFor.

@NgModule({ imports: [BrowserModule] }) // ROOT ONLY — once per app
export class AppModule {}

@NgModule({ imports: [CommonModule] })  // feature modules
export class OrdersModule {}             // gets ngIf/ngFor, not browser services
OutputImport BrowserModule twice and Angular throws. BrowserModule = CommonModule + bootstrap/render services.

SharedModule providers pitfall

Never list stateful services in a SharedModule's providers: each lazy-loaded feature gets its own injector and its own instance.

// BAD — every lazy module that imports this gets a SEPARATE CartService
@NgModule({ exports: [CardComponent], providers: [CartService] })
export class SharedModule {}

// GOOD — one app-wide singleton, SharedModule stays declarations-only
@Injectable({ providedIn: 'root' })
export class CartService {}
OutputKeep services out of shared module providers; use providedIn:'root' so the whole app shares one instance.

Internationalization (i18n)

Mark translatable text i18n

The i18n attribute flags an element's text for translation. It is a build-time marker, not a directive — nothing happens at runtime.

<button i18n>Edit Course</button>
<h4 i18n>Welcome to the course</h4>
OutputBoth strings become translatable units; without extraction the marker does nothing.

Custom translation IDs @@myId

By default Angular hashes an ID per message. Pin a stable key with meaning|description@@customId — every part is optional.

<h4 i18n="@@welcomeMessage">Welcome</h4>

<h4 i18n="welcome|Greeting at course start@@welcomeMessage">
  Welcome to the course
</h4>
OutputThe trans-unit uses welcomeMessage instead of a generated hash; prefer auto IDs, use custom rarely.

Pluralization {n, plural, ...}

ICU plural picks count-dependent wording as one translatable unit. Use =0/=1 for exact matches, other as fallback.

<span i18n>
  { totalCourses, plural,
    =0 {No courses available}
    =1 {One course available}
    other {{{totalCourses}} courses available}
  }
</span>
OutputAll forms extract together; the count becomes an {INTERPOLATION} placeholder. Beware: =1 (exact) ≠ CLDR one (grammar category).

Alternative expressions {k, select, ...}

ICU select chooses one string from string keys — a translation-friendly replacement for ngSwitch, grouped as one unit.

<div class="category" i18n>
  { course.category, select,
    BEGINNER {Beginner}
    INTERMEDIATE {Intermediate}
    ADVANCED {Advanced}
  }
</div>
OutputExtracts as one unit with grouped alternatives; a missing comma after select is a build error.

Extract messages ng extract-i18n

Scans marked templates and writes one message file (default src/messages.xlf) to hand to a translator.

ng extract-i18n --format json
ng extract-i18n --output-path src/locale
OutputA trans-unit per marked string, e.g. <source>Edit Course</source>, ready for translation.

Build per locale localize

Register locales in angular.json's top-level i18n block, then build a separate AOT bundle per locale.

"i18n": {
  "sourceLocale": "en",
  "locales": {
    "fr": { "translation": "src/locale/messages.fr.xlf" }
  }
}

OutputEach locale compiles to dist/<project>/<locale>; set i18nMissingTranslation: "error" for production.

How auto IDs are generated

The auto-hash is derived from the message content plus meaning — the description is only a translator note and does not affect the ID.

<!-- editing the text changes the hash => needs re-translation -->
<h4 i18n>Welcome</h4>          <!-- id A -->
<h4 i18n>Welcome!</h4>         <!-- id B (new) -->

<!-- meaning is part of the hash -->
<h4 i18n="site|">Home</h4>      <!-- id differs from -->
<h4 i18n="menu|">Home</h4>      <!-- ...this one -->
OutputEditing text yields a new ID (old translation dropped). Moving the element or changing description-only keeps the same ID — so a stale translation can silently persist. Use @@customId to pin it.

Driving a custom element (inputs/outputs)

Deferrable views @defer { }

Lazy-loads a template fragment and its dependencies into a separate JS chunk, fetched only when a trigger fires. More granular than route-level lazy loading.

@defer {
  <courses-card-list [courses]="courses"></courses-card-list>
}
Outputmain.js shrinks; card-list ships as chunk-XXXX.js. Default (no trigger) = on idle.

Companion blocks @placeholder / @loading / @error

@placeholder shows before the trigger, @loading while the chunk downloads, @error if it fails. All three load eagerly — keep them light.

@defer {
  <courses-card-list [courses]="courses"></courses-card-list>
} @loading (after 1s; minimum 3s) {
  <h2>Loading...</h2>
} @error {
  <h2>Could not load this section</h2>
} @placeholder (minimum 3s) {
  <p>Placeholder content</p>
}
Outputafter skips the spinner on fast loads; minimum prevents flicker. @placeholder takes only minimum.

Built-in triggers @defer (on …)

Choose when the block displays. viewport/interaction/hover observe the @placeholder by default, or a template-ref element you pass in.

@defer (on idle) { }         <!-- default: browser idle -->
@defer (on immediate) { }    <!-- as soon as it renders -->
@defer (on timer(5s)) { }    <!-- fixed delay -->
@defer (on viewport(title)) { } <!-- #title scrolls into view -->
@defer (on interaction) { } @placeholder { } <!-- click / keydown; needs placeholder -->
@defer (on hover) { } @placeholder { }        <!-- pointer enters; needs placeholder -->
Outputviewport/interaction/hover need a placeholder OR a template-ref target, else compile error.

Prefetch & custom when triggers

prefetch on … downloads on its own clock, separate from display; when <expr> drives either from any truthy component expression. Display always outranks prefetch.

<!-- fetch early on hover, reveal only on click -->
@defer (on interaction(printBtn); prefetch on hover(printBtn)) {
  <pdf-export-dialog></pdf-export-dialog>
} @placeholder { <p>Print-ready export</p> }

<!-- custom: driven by component state -->
@defer (when display; prefetch when performPrefetch) { ... }
OutputChunk arrives on hover while placeholder stays; click swaps in the content instantly (already cached).

Multiple triggers (OR semantics)

Semicolon-separate several display triggers in one @defer (…): whichever fires first wins. Distinct from the display + prefetch combo, which run on separate clocks.

@defer (on idle; on viewport) {
  <courses-card-list [courses]="courses"></courses-card-list>
} @placeholder {
  <p>Loading soon…</p>
}
OutputLoads on browser idle OR when scrolled into view — any one condition satisfies; the rest are ignored.

Signals In Depth

Writable signals signal()

A reactive container. Create with signal(initial), read by calling it, and write with set() — the write is what notifies Angular.

import { signal } from '@angular/core';

const count = signal<number>(0);

count();              // read: 0  (invoke like a function)
count.set(5);         // write a new value
count();              // 5
Output0, then 5. In a template read it as {{ count() }} — bare count renders the function, not the value.

update() & read-only view asReadonly()

update(fn) derives the next value from the current one. asReadonly() returns a Signal<T> (no set/update) to hand out while keeping writes private.

const count = signal(0);

count.update(c => c + 1);        // 1 — from current value

const ro = count.asReadonly();   // Signal<number>
// ro.set(2);  &larr; compile error, no writer
ro();                            // 1
Outputcount() is 1. asReadonly() is a live view of the same signal, not a copy — writes through the original are still seen.

Objects & arrays — replace, never mutate

Signals compare by reference (Object.is). Mutating in place skips the notification, so consumers never update. Always set a new reference.

const course  = signal({ id: 1, title: 'Basics' });
const courses = signal<string[]>(['Basics']);

// WRONG — no notification:
// course().title = 'Hi';  courses().push('Deep Dive');

// RIGHT — new reference via set / update:
course.set({ id: 1, title: 'Hi' });
courses.update(list => [...list, 'Deep Dive']);
Outputtitle = 'Hi'; array = ['Basics','Deep Dive']. In-place mutation may "work" under Zone.js but breaks in a zoneless setup.

Derived signals computed()

Builds a read-only signal from other signals. It recomputes automatically when a source changes, and is lazy + memoized (only recalculates on read, when a source actually moved).

import { signal, computed } from '@angular/core';

const counter = signal(0);
const derived = computed(() => counter() * 10);  // read-only

derived();          // 0
counter.set(3);
derived();          // 30 — recomputed on demand
Output0, then 30. A computed has no set()/update() — change its output only by changing a source.

Automatic dependency tracking

Angular records which signals a computed/effect reads while it runs. Tracking is dynamic — a source read inside a skipped if forms no dependency. Read sources unconditionally.

// BROKEN: counter() only read when branch runs
const broken = computed(() => {
  if (flag) { return counter() * 10; }  // dep not registered
  return 0;
});

// FIXED: read the source first, always
const fixed = computed(() => {
  const v = counter();                  // dep always registered
  return flag ? v * 10 : 0;
});
Outputbroken stops reacting to counter; fixed recomputes on every counter change.

Side effects effect()

Runs a function reactively when a signal it reads changes — for logging, storage, network. Runs once up front to record deps; multiple writes in one tick coalesce into a single run.

import { signal, effect } from '@angular/core';

const counter = signal(0);
const name = signal('Ada');

effect(() => console.log(`${name()}: ${counter()}`));

counter.set(1);
name.set('Grace');   // both changes in one tick
Output"Ada: 0", then "Grace: 1" logged once. Don't write signals inside an effect (throws by default) — reserve for true side effects. Create in an injection context (constructor).

Effect cleanup onCleanup / EffectRef.destroy()

Effects auto-destroy with their component. For manual control pass { manualCleanup: true }; the returned EffectRef.destroy() stops it. The onCleanup param runs teardown before each re-run and on destroy.

const ref = effect((onCleanup) => {
  const v = counter();
  const id = setInterval(() => console.log('tick', v), 1000);
  onCleanup(() => clearInterval(id));   // release the timer
}, { manualCleanup: true });

ref.destroy();   // stops the effect; runs final onCleanup
OutputTicks each second until destroy(), which clears the interval once. Note: no OnCleanup interface exists — it's the lowercase callback param.

Signal-based data services

Share a signal across unrelated components via an injectable. Keep the writable signal private, expose asReadonly(), and offer methods as the only mutation path.

@Injectable({ providedIn: 'root' })
export class CounterService {
  private counter = signal(0);                  // private, writable
  readonly value = this.counter.asReadonly();   // read-only view

  increment() {
    if (this.counter() >= 10) throw new Error('Max reached');
    this.counter.update(v => v + 1);            // controlled mutation
  }
}
OutputConsumers read value() and call increment() but cannot set(). Centralizes validation; use only for genuinely shared state.

Signal inputs input()

Declares a component input that is a signal (InputSignal<T>). Read it by calling it, and it composes directly with computed()/effect(). Read-only by design.

import { Component, input, computed } from '@angular/core';

@Component({ selector: 'course-card', standalone: true,
  template: `<h3>{{ label() }}</h3>` })
export class CourseCardComponent {
  course = input<Course>();   // InputSignal<Course | undefined>
  label = computed(() => this.course()?.title ?? '');
}
OutputRenders the bound course's title; label() recomputes whenever the parent passes a new value. (For two-way binding use model().)

required, alias, transform input.required()

input.required<T>() makes a binding mandatory (no default). alias renames the binding; transform maps the incoming value before it is stored.

export class CourseCardComponent {
  title = input.required<string>();          // must be bound

  course = input<number>(0, {
    alias: 'tutorial',                        // bind as [tutorial]
    transform: (v: number) => v * 100,
  });
}
// <course-card [title]="'Angular'" [tutorial]="2">
Outputtitle() = 'Angular', course() = 200. Built-ins booleanAttribute/numberAttribute coerce string-ish template values.

Replace ngOnChanges with a signal input

A decorator input pushes you to ngOnChanges to observe changes. Read a signal input inside effect() (or computed()) instead and drop the hook entirely.

// BEFORE: decorator input + lifecycle hook
class C implements OnChanges {
  @Input() course!: Course;
  ngOnChanges() { console.log('changed', this.course); }
}

// AFTER: signal input + effect, no ngOnChanges
class C {
  course = input<Course>();
  constructor() { effect(() => console.log('changed', this.course())); }
}
OutputLogs on every new course from the parent. Prefer computed() for derived values; reserve effect() for real side effects.

Two-way signal binding model()

Pairs a writable signal input with an auto-generated ...Change output, enabling banana-in-a-box [(x)]. Write to it in the child; the parent stays in sync.

import { Component, model } from '@angular/core';

@Component({ selector: 'counter', standalone: true,
  template: `<button (click)="inc()">{{ count() }}</button>` })
export class CounterComponent {
  count = model(0);                 // ModelSignal<number>
  inc() { this.count.update(c => c + 1); }  // writes flow to parent
}
// parent: <counter [(count)]="total"></counter>
OutputClicking updates both child and parent total. [(count)] desugars to [count] + (countChange).

Custom equality { equal }

Override the default Object.is with an equal comparator on signal()/computed() to suppress redundant notifications when a structurally-equal object is set.

const point = signal(
  { x: 0, y: 0 },
  { equal: (a, b) => a.x === b.x && a.y === b.y },
);

point.set({ x: 0, y: 0 });   // deep-equal &rarr; no notification
point.set({ x: 1, y: 0 });   // differs &rarr; notifies consumers
OutputFirst set is a no-op; only the second fires. Useful for the objects/arrays case where a new reference holds the same data.

Writing signals in effects allowSignalWrites

Version nuance: in v17–18 writing a signal inside an effect throws unless you pass { allowSignalWrites: true }. In v19+ the flag is removed — effects allow writes by default.

// v17-18: escape hatch required
effect(() => {
  const v = source();
  mirror.set(v);                    // throws without the flag
}, { allowSignalWrites: true });

// v19+: no flag — writes allowed by default
effect(() => mirror.set(source()));
OutputEither way mirror tracks source. Prefer computed() for derived state; writing in effects risks change-detection loops.

results$ = this.query.valueChanges.pipe(
